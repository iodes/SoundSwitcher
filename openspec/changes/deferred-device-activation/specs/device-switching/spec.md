## MODIFIED Requirements

### Requirement: 선호 장치 순환 전환 (사용자 지정 순서)
전환 명령이 발생하면, 시스템은 사용자가 배치한 순서에 따라 다음 '프로파일'에 지정된 입출력 장치로 기본 장치를 변경해야 한다. 단, 전환 대상이 되는 프로파일에 설정된 장치 중 하나라도 연결이 끊어졌거나 사용할 수 없는 상태(비활성화 등)인 경우, 해당 프로파일로의 전환을 스킵하지 않고 대기(Pending) 상태로 진입하여 장치가 활성화될 때까지 기다려야 한다.

#### Scenario: 지정된 순서대로 프로파일 전환
- **WHEN** 사용자가 '헤드셋 프로파일 -> 스피커 프로파일' 순서로 배치한 상태에서 전환 명령이 발생할 때
- **THEN** 활성화 상태가 '헤드셋 프로파일'에서 '스피커 프로파일'로 넘어가고, 해당 프로파일에 설정된 입출력 장치가 모두 기본 장치로 변경되어야 한다

#### Scenario: 연결이 끊긴 장치가 포함된 프로파일 대기
- **WHEN** 사용자가 '스피커 프로파일 -> 헤드셋 프로파일(연결 끊김)' 순서로 배치한 상태에서 스피커 프로파일 사용 중 전환 명령이 발생할 때
- **THEN** 헤드셋 프로파일의 장치가 사용할 수 없는 상태이므로 시스템은 헤드셋 프로파일을 대기 상태로 두고 장치가 연결될 때까지 기다려야 한다

### Requirement: 즉각적인 장치 전환 알림 제공
시스템은 장치 전환이 발생했을 때 전환된 장치 정보를 담은 시각적 알림(Notification)을 화면에 표시해야 한다. 단, 알림은 사용자가 프로파일을 '선택/요청'한 시점이 아니라, 실제로 시스템의 기본 오디오 장치가 '성공적으로 전환 완료된 시점'에 표시되어야 한다. 이 알림은 OS의 기본 큐(Queue)에 의존하지 않고 즉각적으로 나타나야 하며, 사용자가 연속으로 장치를 전환할 경우 알림이 중첩되거나 지연되지 않도록 이전 알림을 즉시 덮어쓰거나 닫아야 한다.

#### Scenario: 단일 장치 전환 성공 시 알림 발생
- **WHEN** 사용자가 전환을 요청한 대상 장치가 유효하여 시스템 장치가 실제로 변경되었을 때 (또는 대기 중이던 장치가 활성화되어 자동 전환에 성공했을 때)
- **THEN** 시스템은 전환된 장치 이름과 아이콘을 포함한 알림 창을 즉시 표시해야 한다

#### Scenario: 장치 전환 실패 또는 대기 상태 진입 시 알림 생략
- **WHEN** 사용자가 프로파일 전환을 요청했으나 대상 장치가 사용할 수 없는 상태여서 전환에 실패하거나 '대기(Pending)' 상태로 진입했을 때
- **THEN** 실제 장치 전환이 이루어지지 않았으므로 전환 알림 창은 표시되지 않아야 한다
