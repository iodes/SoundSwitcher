## Why
사용자가 현재 사용할 수 없는 오디오 장치(예: 연결이 끊긴 블루투스 헤드셋)가 포함된 프로파일로 전환을 시도할 수 있습니다. 현재 이 경우 전환이 즉시 실패하거나 스킵됩니다. 장치가 연결될 때까지 장치 전환을 대기(Defer)시키면, 사용자가 먼저 장치 전환을 예약하고 나중에 기기를 켤 수 있어 사용자 경험(UX)이 향상됩니다.

## What Changes
- 대상 입력 또는 출력 장치가 비활성/연결 해제 상태인 프로파일로 전환을 시도할 때, 전환 작업이 보류(대기) 상태가 됩니다.
- 메인 화면의 프로파일 카드(Profile Card) 아이콘 우측 하단에 표시되던 기존의 초록색 활성 체크 마크 대신 로딩 스피너가 표시되어 전환 대기 중임을 알립니다.
- 시스템에 불필요한 부하를 주지 않기 위해(폴링 방식 배제) Windows Core Audio API의 장치 상태 변경 알림과 같은 CPU 효율적인 메커니즘을 사용하여 장치 상태를 모니터링합니다.
- 대상 장치가 활성화되면 자동으로 예약된 프로파일 전환이 실행됩니다.
- 장치 전환 알림(Notification)이 프로파일을 선택(요청)한 시점이 아닌, 대상 장치가 성공적으로 활성화되어 실제 전환이 완료된 시점에만 표시되도록 알림 트리거 로직을 공통화하고 수정합니다.

## Capabilities

### New Capabilities
- `deferred-device-activation`: 프로파일 전환을 큐에 넣고 대상 장치가 활성화될 때까지 효율적으로 대기하는 기능을 처리합니다.

### Modified Capabilities
- `device-switching`: 장치 전환 로직에서 즉시 실패하거나 스킵하는 대신 대기(Pending) 상태를 반환하고 예약된 실행을 처리하도록 수정해야 합니다. 또한 전환 알림이 전환 요청 시가 아닌 실제 전환 성공 시에만 트리거되도록 수정합니다.
- `profile-management`: 프로파일 카드의 아이콘 오버레이 요소에서 활성화 상태의 초록색 체크 마크 대신 "대기 중" 상태를 나타내는 로딩 스피너 애니메이션을 지원하도록 수정해야 합니다.
- `tray-interface`: 트레이 아이콘은 장치 대기 상태 진입 시 아이콘 이미지 변경 없이, 마우스 오버 툴팁(Tooltip) 텍스트를 통해 전환 대기 중임을 표기하도록 수정해야 합니다.

## Impact
- 오디오 장치 전환 핵심 로직(`AudioDeviceManager` 등)에서 지연된 전환 기능 지원.
- 프로파일 카드 UI 컴포넌트(`ProfileCard` 등)를 수정하여 로딩 스피너 표시.
- 폴링 대신 CPU 효율적인 모니터링을 보장하기 위해 C#에서 `IMMNotificationClient` 또는 유사한 장치 상태 변경 리스너 구현.
