## Context

현재 시스템 트레이 아이콘의 좌클릭 동작은 다음 프로파일로 전환되도록 하드코딩되어 있습니다. 그러나 이 로직은 사용자가 프로파일을 전혀 구성하지 않았거나 단 하나의 프로파일만 구성한 경우와 같은 엣지 케이스를 적절히 처리하지 못합니다. 프로파일이 0개인 경우 클릭해도 아무런 반응이 없어 메인 창으로의 접근을 어렵게 합니다. 프로파일이 1개인 경우에는 변경할 대상이 없음에도 불구하고 불필요한 프로파일 전환 로직이 실행됩니다.

## Goals / Non-Goals

**Goals:**
- 프로파일이 0개일 때 트레이 아이콘을 좌클릭하면 메인 창이 열려야 합니다.
- 구성된 프로파일이 1개뿐인 경우 프로파일 전환 동작을 조기에 중단(return)해야 합니다.

**Non-Goals:**
- 프로파일 관리자나 트레이 상호작용 시스템 전체를 리팩토링하는 것은 제외합니다.
- 이러한 엣지 케이스에 대한 추가적인 UI 알림은 제외합니다 (단순히 자연스럽게 동작하도록 구현).

## Decisions

- **트레이 클릭 핸들러 수정**: 트레이 클릭 이벤트 핸들러(주로 `App.xaml.cs`에 위치)의 로직을 직접 수정합니다.
- **조건문 분기**: `ProfileManager.Profiles.Count` 값을 평가합니다.
  - `Count == 0`인 경우, 기존의 `ShowMainWindow` 관련 명령이나 메서드를 호출합니다.
  - `Count == 1`인 경우, 현재 시스템에 활성화된 프로파일인지 확인하여, 이미 활성화된 상태라면 불필요한 전환 알림이 뜨지 않도록 즉시 함수를 빠져나옵니다(return). 활성화되지 않은 상태라면 해당 프로파일을 선택합니다.
  - `Count > 1`인 경우, 기존과 같이 다음 프로파일로 전환하는 로직을 수행합니다.

## Risks / Trade-offs

- **Risk**: 메인 창을 띄우는 로직을 재사용 가능한 메서드로 추출하지 않으면 코드 중복이 발생할 수 있습니다.
- **Mitigation**: 기존에 구현된 `ShowWindow` 또는 `MainWindow.Show()` 명령/메서드를 적절히 호출하여 코드 중복을 방지합니다.
