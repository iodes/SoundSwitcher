## Context

현재 사운드 스위처 애플리케이션은 핫키를 통해 프로파일을 변경할 때 알림(Notification)을 표시합니다. 그러나 메인 UI에서 프로파일 카드를 클릭하여 직접 전환할 때는 이 알림이 발생하지 않아 사용자 경험의 일관성이 떨어집니다. 또한 사용자들 중에는 프로파일 변경 알림 자체가 방해된다고 느껴 이를 비활성화하고 싶어하는 요구사항이 있을 수 있습니다. 이에 알림 로직을 공통화하고 설정을 추가하는 작업이 필요합니다.

## Goals / Non-Goals

**Goals:**
- 프로파일 변경 알림 표시 로직을 재사용 가능한 형태로 분리(공통화)합니다.
- 메인 윈도우 UI에서 프로파일을 클릭하여 전환할 때도 알림이 표시되도록 연동합니다.
- 사용자가 설정 메뉴에서 프로파일 변경 알림 표시 여부를 켜고 끌 수 있는 옵션을 제공합니다.

**Non-Goals:**
- 알림 UI(디자인/애니메이션 등) 자체를 리팩토링하거나 크게 변경하는 것은 포함하지 않습니다. (단지 표시 여부와 호출 시점만 수정)
- 다른 종류의 알림(에러 메시지 등) 처리는 본 작업의 범위에서 제외됩니다.

## Decisions

- **알림 표시 설정 추가 (`ShowProfileNotification` 또는 유사한 이름):** 설정 모델(`Settings` 관련 클래스)에 boolean 타입의 속성을 추가하고 기본값은 `true`로 설정합니다.
  - **Rationale:** 기존 사용자의 기본 경험을 유지하면서 필요시 사용자가 옵션을 끄도록 하기 위함입니다.
- **알림 로직 공통화 및 캡슐화:** 알림을 호출하는 서비스 또는 유틸리티(`NotificationService` 등) 내부에서 설정 값을 체크하는 로직을 포함시켜 호출부에서 별도로 설정을 확인하지 않도록 합니다.
  - **Rationale:** 핫키 이벤트 핸들러와 프로파일 카드 마우스 클릭 핸들러 각각에서 설정을 확인하는 중복 코드를 방지할 수 있습니다.
- **UI 연동:** `MainViewModel` (또는 프로파일 선택 처리 로직)에서 프로파일 활성화 처리가 완료된 직후 공통 알림 메서드를 호출하도록 수정합니다.

## Risks / Trade-offs

- **[Risk] 여러 경로에서 동시 전환 발생:** 메인 UI에서 클릭하고 거의 동시에 핫키를 누르는 등의 동작 시 알림이 중복되거나 연속으로 스택될 수 있습니다.
  → **Mitigation:** 알림 시스템 자체의 기존 중복 방지/대체 큐 로직에 의존하며, 본 작업에서는 호출을 공통화하는 것에 집중합니다.
