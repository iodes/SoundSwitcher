## Context

현재 SoundSwitcher는 사용자가 명시적으로 선택한 프로파일로 오디오 장치 설정을 변경합니다. 하지만 사용자는 컴퓨터를 시작할 때 항상 고정된 특정 프로파일(기본 프로파일)이 설정되어 있기를 원할 수 있습니다. 이전 종료 전에 다른 프로파일을 사용했더라도 시작 시에는 항상 기본 프로파일로 초기화되어 편의성을 높일 필요가 있습니다.

## Goals / Non-Goals

**Goals:**
- 시스템 트레이 메뉴의 프로파일 항목에서 우클릭 시 "기본 프로파일로 설정" 및 "기본 프로파일 설정 해제" 옵션 제공 (아이콘 변경 항목 위에 위치, 다국어 지원)
- 다른 프로파일을 기본 프로파일로 설정하면 기존 기본 프로파일은 자동으로 해제되어 단일 기본 프로파일만 유지되도록 함
- 앱 시작(윈도우 시작) 시 설정된 기본 프로파일이 있고, 현재 설정된 오디오 상태가 기본 프로파일과 다를 경우 자동으로 기본 프로파일을 적용
- 프로파일 비교(동일성 판정)는 기존의 공용 로직을 재사용하여 정확성과 일관성 유지

**Non-Goals:**
- 여러 개의 기본 프로파일 지정 기능
- 앱 실행 도중에 자동으로 프로파일을 롤백하는 기능 (앱 시작 시점에 1회만 동작)

## Decisions

1. **설정 저장소 (Settings)**: 
   앱의 글로벌 설정 파일에 `DefaultProfileId` (string/guid 등) 필드를 추가하여 사용자가 선택한 기본 프로파일을 기억합니다. 코드 구조상 이 속성은 `lastSelectedProfileId` 바로 위에 위치하도록 배치합니다. 다른 프로파일을 기본으로 설정하면 이 값을 덮어쓰고, 설정 해제하면 null 또는 빈 문자열로 초기화합니다.
2. **UI 컨텍스트 메뉴**:
   프로파일 항목 우클릭 메뉴의 MenuItems 중간에 "기본 프로파일로 설정" (또는 "기본 프로파일 설정 해제") MenuItem을 삽입합니다. (위치: 아이콘 변경 위) 해당 메뉴 항목은 다국어 리소스를 사용해야 합니다. 현재 프로파일이 기본 프로파일인지 여부에 따라 "해제" 메뉴 또는 체크 상태를 보여줄 수 있습니다.
3. **시작 시 자동 전환 로직**:
   앱이 시작되고 트레이 메뉴 및 초기화가 완료될 무렵, 현재 활성화된 오디오 디바이스 맵핑 상태와 기본 프로파일에 정의된 상태를 비교합니다. 비교 시 기존에 프로파일 활성화 여부를 체크하는 공용 로직을 재사용하여 다를 경우 프로파일 적용 메서드를 호출합니다.

## Risks / Trade-offs

- **Risk**: 사용자가 기본 프로파일로 지정한 프로파일을 나중에 삭제하거나 이름을 변경할 수 있음.
  **Mitigation**: 앱 시작 시 `DefaultProfileId`로 프로파일 목록을 검색하고, 유효한 프로파일 객체를 찾을 수 없는 경우(이미 삭제됨 등) 전환 로직을 건너뜁니다.
