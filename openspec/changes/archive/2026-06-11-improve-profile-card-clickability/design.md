## Context

SoundSwitcher의 메인 윈도우에는 사용자가 구성한 오디오 프로파일 목록이 카드 형태로 나열됩니다. 기존 설계에서는 프로파일 카드의 아이콘 영역을 직접 클릭하여 커스텀 아이콘으로 변경하는 기능이 있었습니다. 그러나 이는 카드를 클릭하여 프로파일을 전환하려는 사용자의 주된 조작 의도를 방해하고, 카드의 유효 클릭 영역(프로파일 전환용)을 좁히는 원인이 되었습니다. 더불어 카드가 클릭 가능한 요소임을 나타내는 시각적 힌트(마우스 오버나 클릭 시 상태 변화)가 부족하여 UX 측면의 개선이 요구됩니다.

## Goals / Non-Goals

**Goals:**
- 프로파일 카드 전반에 마우스 오버(Hover) 및 마우스 클릭(Mouse Down) 시 시각적 애니메이션 및 스타일 피드백(크기 축소, 배경색 변경 등) 적용
- 아이콘 클릭을 통한 아이콘 변경 진입점을 제거하여, 아이콘 영역까지 프로파일 활성화 클릭 영역으로 편입
- 프로파일 카드의 우클릭 컨텍스트 메뉴에 "아이콘 변경" 항목 추가

**Non-Goals:**
- 프로파일 카드의 근본적인 레이아웃 및 디자인 테마 전면 교체
- 커스텀 아이콘을 복사하고 캐시하는 백엔드 파일 입출력 로직 수정 (기존 로직 재사용)

## Decisions

- **시각적 클릭/호버 피드백 추가:**
  - `ProfileCard.xaml`에 정의된 최상위 Border 혹은 관련된 Style에 WPF Trigger를 적용합니다. `IsMouseOver` 시 背景 색상을 미세하게 밝게 만들고, `MouseLeftButtonDown` 또는 커스텀 상태 변경 시 `ScaleTransform`을 이용해 카드가 약간 줄어드는 애니메이션 피드백을 제공합니다.
  - *Rationale*: 직관적인 클릭 피드백을 통해 버튼처럼 반응하는 UI 경험을 사용자에게 제공할 수 있습니다.
- **아이콘 변경 진입점 이동:**
  - 기존 아이콘 이미지 컨트롤에 연결되어 있던 클릭(MouseLeftButtonDown 또는 Button) 이벤트 핸들러를 제거합니다. 
  - `ProfileCard`의 `ContextMenu`에 "아이콘 변경"(`MenuItem`)을 추가하고, 기존의 아이콘 파일 선택 및 변경 로직을 해당 MenuItem의 `Click` 이벤트 핸들러로 이동시킵니다.
  - *Rationale*: 설정 변경과 관련된 액션들을 우클릭 메뉴로 응집시킴으로써 사용자가 실수로 아이콘 설정을 건드리는 일을 방지하고 카드 클릭 영역을 확보할 수 있습니다.
- **프로파일 활성화 클릭 영역 확장:**
  - 아이콘 컨트롤 자체 이벤트를 제거함에 따라, 자연스럽게 아이콘 위에서의 클릭 이벤트도 부모 컨테이너로 버블링되어 프로파일 활성화 이벤트로 처리되도록 합니다. 
  - *Rationale*: 사용자가 고민 없이 카드의 넓은 영역을 탭하여 장치를 전환할 수 있게 합니다.

## Risks / Trade-offs

- [Risk] 이벤트 라우팅 문제로, 드래그 핸들이나 콤보박스를 클릭할 때도 카드 클릭(축소 애니메이션 등) 효과가 발생하거나 프로파일이 전환될 수 있습니다.
  - *Mitigation*: 내부 컨트롤(콤보박스, 드래그 썸 등)을 조작할 때는 마우스 이벤트를 `e.Handled = true`로 처리하여 버블링을 차단하고 클릭 효과가 적용되지 않도록 세밀하게 제어합니다.
