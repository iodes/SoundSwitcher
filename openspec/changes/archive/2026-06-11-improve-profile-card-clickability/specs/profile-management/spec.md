## ADDED Requirements

### Requirement: 프로파일 카드 상호작용 시각적 피드백
프로파일 카드는 클릭 가능한 요소임을 사용자에게 명확히 전달하기 위해 마우스 상호작용에 따른 시각적 피드백을 제공해야 한다.

#### Scenario: 마우스 오버 시 시각적 피드백
- **WHEN** 사용자가 마우스 커서를 프로파일 카드 위로 올릴 때
- **THEN** 프로파일 카드의 배경색이 미세하게 밝아지거나 어두워져 마우스 오버 상태임을 나타내야 한다

#### Scenario: 마우스 클릭(다운) 시 시각적 피드백
- **WHEN** 사용자가 프로파일 카드를 마우스로 좌클릭(다운)할 때
- **THEN** 프로파일 카드의 크기가 미세하게 줄어들거나 눌린듯한 시각적 효과가 발생하여 클릭 동작이 인지되어야 한다

## MODIFIED Requirements

### Requirement: 프로파일 카드 아이콘 표시 및 설정
프로파일 카드는 드래그 재정렬 그립 옆에 아이콘을 표시해야 한다. 유저가 별도로 커스텀 아이콘을 지정하지 않으면 해당 프로파일에 설정된 **재생 장치(Playback Device)**의 시스템 아이콘을 기본값으로 사용해야 한다. 사용자는 컨텍스트 메뉴를 통해 커스텀 이미지(ICO, PNG, JPEG)로 아이콘을 변경할 수 있어야 한다. 설정된 커스텀 이미지는 원본 파일 위치에 상관없이 유지되도록 앱 데이터 폴더 내에 캐싱되어야 한다.

#### Scenario: 기본 재생 장치 아이콘 표시
- **WHEN** 사용자가 커스텀 아이콘을 설정하지 않았거나 '기본 아이콘으로 초기화'를 수행한 상태일 때
- **THEN** 프로파일 카드 내에 해당 프로파일에 매핑된 재생 장치의 Windows 시스템 아이콘이 자동으로 렌더링되어야 한다

#### Scenario: 커스텀 아이콘 선택 및 캐싱
- **WHEN** 사용자가 해당 카드의 우클릭 컨텍스트 메뉴에서 '아이콘 변경' 항목을 좌클릭하여 지원되는 이미지 파일(ICO, PNG, JPEG)을 선택할 때
- **THEN** 시스템은 선택된 이미지를 앱 데이터 폴더로 복사/캐싱하고, 설정에는 원본의 절대 경로 대신 캐시된 아이콘 경로를 저장하여 아이콘 렌더링에 사용해야 한다

### Requirement: 프로파일 카드 클릭을 통한 활성화
사용자는 프로파일 카드의 일반 영역(아이콘 영역 포함, 단 콤보박스 및 드래그 핸들 제외)을 좌클릭하여 해당 프로파일을 활성화(전환)할 수 있어야 한다.

#### Scenario: 일반 영역 및 아이콘 영역 좌클릭을 통한 전환
- **WHEN** 사용자가 프로파일 카드의 정보가 표시되는 일반 영역 혹은 아이콘 영역을 좌클릭할 때
- **THEN** 해당 카드의 프로파일이 즉시 활성화(선택)되고 사운드 재생 장치가 해당 프로파일에 지정된 장치로 전환되어야 한다

#### Scenario: 내부 컨트롤과 상호작용 시 전환 방지
- **WHEN** 사용자가 프로파일 카드 내부의 연결 장치 선택 콤보박스, 혹은 좌측 드래그 핸들을 좌클릭할 때
- **THEN** 각각의 컨트롤 고유 기능만 수행되며, 프로파일 전환 이벤트는 발생하지 않아야 한다

### Requirement: 컨텍스트 메뉴를 통한 프로파일 관리 기능
프로파일 카드 내의 오동작 방지 및 기능 응집을 위해, 삭제, 아이콘 변경, 기본 아이콘으로 초기화 기능은 카드 전체 영역에 대한 우클릭 컨텍스트 메뉴를 통해 제공되어야 한다.

#### Scenario: 삭제 및 관리 컨텍스트 메뉴 호출
- **WHEN** 사용자가 프로파일 카드 위에서 우클릭할 때
- **THEN** 프로파일 카드 위치 근처에 '삭제', '아이콘 변경' 등 관리 옵션이 포함된 컨텍스트 메뉴가 나타나야 한다

#### Scenario: 메뉴를 통한 삭제 실행 및 캐시 삭제
- **WHEN** 사용자가 컨텍스트 메뉴의 '삭제' 옵션을 좌클릭할 때
- **THEN** 해당 프로파일이 목록에서 즉시 제거되며, 해당 프로파일이 커스텀 아이콘을 사용 중이었다면 앱 데이터 폴더 내의 연관된 캐시 이미지 파일도 물리적으로 삭제되어야 한다
