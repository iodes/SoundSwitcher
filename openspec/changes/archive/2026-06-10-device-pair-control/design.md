## Context

현재 애플리케이션은 출력 장치와 입력 장치를 완전히 독립적으로 관리하고 있습니다. 그러나 다수의 사용자가 화상 회의, 게임 등 특정 상황에 맞추어 헤드셋(마이크+스피커)으로, 혹은 일반 데스크톱 모드(별도 마이크+스피커)로 입력과 출력을 동시에 변경하는 패턴을 가지고 있습니다. 기존 방식은 두 번의 독립적인 설정 과정이 필요해 불편함이 큽니다.

## Goals / Non-Goals

**Goals:**
- 장치 탭 UI를 전면 개편하여, 사용자가 '출력 장치 + 입력 장치 + 아이콘 + 이름'을 포함하는 **장치 프로파일(페어)**을 생성/편집/삭제할 수 있도록 구성.
- 트레이 좌클릭 시 개별 장치가 아닌 저장된 '프로파일' 단위로 순환 전환되도록 로직 개선.
- 기존 사용자의 설정 파일(JSON)의 버전 관리 및 마이그레이션은 고려하지 않음 (아직 배포되지 않은 프로그램이므로 설정 포맷 변경 시 하위 호환성 무시).

**Non-Goals:**
- 복잡한 조건(특정 프로그램 실행 시 자동 전환 등) 기반의 프로파일 자동 전환 기능.
- 개별 장치 단위의 전환 UI 유지(프로파일 방식으로 완전 대체).

## Decisions

- **Data Model Revision**: 기존 `List<string> PreferredPlaybackIds` 형식의 독립 모델을 버리고, `List<DeviceProfile>` 모델 도입.
  - `DeviceProfile` 클래스 구조: `Id`(GUID), `Name`(프로파일 명), `IconPath`, `PlaybackDeviceId`, `CaptureDeviceId`.
- **UI UX**:
  - 장치 탭 하단에 점선 테두리(Dashed Border)의 "새 프로파일 추가 +" 버튼을 제공하여, 누를 때마다 위쪽에 카드 형태의 페어가 스택처럼 쌓이고 버튼은 아래로 밀려나도록 구성.
  - 프로파일 목록은 카드 리스트로 보여주고, 각 카드 내에 출력 장치와 입력 장치 정보를 아름답게 표시.
  - 각 카드의 좌측에는 네이티브 WPF UI 그립(Handle Grip)을 배치하여 드래그 앤 드롭으로 카드(페어)의 순서를 직관적으로 변경할 수 있게 지원.
  - 체크박스는 완전히 제거하고, 기존 체크박스 위치(또는 적절한 위치)에 마우스 오버 시 나타나는 '휴지통' 모양의 삭제 버튼을 제공.
  - 별도의 수동 새로고침 버튼을 제거하고, Windows 오디오 장치 연결 및 끊김 이벤트를 감지하여 장치 목록을 자동으로 갱신하도록 구성.
- **Switching Logic**: 프로파일 전환 시, 설정된 `PlaybackDeviceId`가 연결되어 있지 않은 경우 예외 처리(해당 장치 변경 스킵), `CaptureDeviceId`도 개별 검사하여 가능한 것만 변경.

## Risks / Trade-offs

- [Risk] 사용자가 프로파일에 설정해 둔 특정 입력 장치가 연결 해제되었을 때, 출력 장치만이라도 변경할 것인가? → [Mitigation] 일부 장치가 누락되었더라도 연결된 장치에 대해서는 변경을 시도하고, 전환 후 툴팁 등을 통해 연결 상태를 반영함.
