## Context

현재 SoundSwitcher 앱은 사용자가 UI에서 특정 프로파일 카드를 클릭하면, 해당 프로파일 ID를 내부 상태로 저장하고 UI에 선택 표시를 띄웁니다. 그러나 OS(Windows) 시스템 설정이나 다른 외부 앱에서 오디오 장치를 변경했을 때는 SoundSwitcher가 이를 즉시 감지하고 UI를 갱신하지 않아 실제 장치와 일치하지 않는 프로파일에 선택 표시가 남아 있는 UI/상태 불일치 문제가 있습니다.

## Goals / Non-Goals

**Goals:**
- UI에 표시되는 "현재 선택된 프로파일" 상태가 항상 실제 시스템 오디오 장치 상태와 동기화되도록 합니다.
- 프로파일 선택 판단 기준을 '마지막 클릭 ID'에서 '시스템 활성 입출력 장치 ID와의 비교'로 변경합니다.
- 외부 오디오 장치 변경 이벤트를 감지하여 프론트엔드의 선택 상태를 자동 갱신합니다.
- 프로파일 콤보박스 변경 시 활성 상태를 즉시 재평가하고, 이에 따라 트레이 아이콘이 실시간으로 동기화되도록 합니다.

**Non-Goals:**
- 프로파일 생성/수정/삭제 등 장치 짝짓기(Pairing) 기능의 기존 로직 자체는 변경하지 않습니다.
- 새로운 UI 디자인 요소를 추가하는 것이 아니며 기존의 "초록색 체크" 표시 조건을 변경하는 데 집중합니다.

## Decisions

1.  **프로파일 검증 조건 변경**
    -   변경 후 (다중 조건 결합):
        1. **필수 조건 (System Match):** `profile.outputDeviceId === currentSystemOutputId && profile.inputDeviceId === currentSystemInputId`
        2. **식별 조건 (Profile ID Tie-breaker):** 위 필수 조건을 만족하는 프로파일이 여러 개일 경우, `profile.id === lastSelectedProfileId` 인 프로파일을 우선적으로 '선택' 상태로 판별합니다.
        3. **주의점:** `lastSelectedProfileId` 와 일치하는 프로파일이더라도, 현재 시스템의 장치와 어긋나 있으면(필수 조건 불만족) 무조건 선택 상태(초록색 체크)를 해제합니다.
    -   **Rationale:** 상태를 단일 진실 공급원(Single Source of Truth, 즉 실제 OS 장치 상태)에 의존하게 만들되, 동일 장치 쌍을 가진 프로파일들을 명확히 구분할 수 있도록 기존 `lastSelectedProfileId`를 보조 지표로 활용합니다.

2.  **OS 오디오 장치 변경 이벤트 구독 및 갱신**
    -   Windows Core Audio API의 `OnDefaultDeviceChanged` 등 오디오 장치 변경 이벤트를 백엔드에서 구독합니다.
    -   해당 이벤트 발생 시 프론트엔드로 메시지를 보내 현재 시스템 입/출력 장치 ID 상태를 업데이트하고, 이 상태에 기반해 어떤 프로파일 카드를 활성화 상태로 보여줄지 리렌더링합니다.
    -   **Rationale:** 사용자가 앱을 조작하지 않더라도 상태가 최신으로 유지되어야 하므로, 주기적 폴링(Polling) 대신 리소스 낭비가 적은 이벤트 기반(Event-driven) 갱신이 적합합니다.

## Risks / Trade-offs

-   **Risk:** 여러 프로파일이 동일한 입출력 장치 쌍을 가지고 있을 경우, 장치가 일치하는 프로파일이 여러 개가 되어 어떤 프로파일에 초록색 체크가 떠야 할지 모호해짐
    -   **Mitigation:** `lastSelectedProfileId` 상태를 계속 유지 및 활용하여, 장치 쌍이 모두 일치하는 프로파일들 중 사용자가 마지막으로 직접 클릭했던 프로파일 ID에 우선적으로 체크를 표시하도록 로직을 고도화합니다.
-   **Risk:** 잦은 장치 변경 이벤트 발생으로 인한 프론트엔드 잦은 리렌더링
    -   **Mitigation:** 백엔드에서 이벤트를 보낼 때 이전 활성 장치 ID와 달라진 경우에만 이벤트를 방출하거나 디바운스(Debounce) 처리를 추가합니다.
