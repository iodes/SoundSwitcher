## Why

현재 프로파일 선택 상태는 단순 프로파일 ID 만으로 관리되고 있어, 사용자가 외부(다른 프로그램이나 Windows 설정 등)에서 직접 장치를 변경할 경우 실제 시스템 장치와 선택된 프로파일 간의 불일치가 발생합니다. 이로 인해 잘못된 UI 피드백(실제 장치가 다른데도 특정 프로파일이 선택된 것처럼 초록색 체크 표시 유지)이 제공됩니다. 이를 해결하기 위해 실제 입출력 장치와 프로파일의 장치를 비교하는 검증 로직 강화가 필요합니다.

## What Changes

- 현재 선택된 프로파일 판단 로직을 프로파일 ID 기반에서 **프로파일 ID + 실제 활성화된 입/출력 장치 ID 비교** 방식으로 변경합니다.
- 외부 요인(OS 설정, 다른 앱 등)에 의한 오디오 장치 변경 이벤트를 감지하여 현재 선택된 프로파일 상태를 즉시 갱신합니다.
- 프로파일 내의 장치 설정(콤보박스)을 변경할 경우 즉각적으로 선택 상태를 재평가하고, 트레이 아이콘이 항상 현재 활성 프로파일에 맞게 동기화되도록 개선합니다.

## Capabilities

### New Capabilities
- `profile-state-sync`: 오디오 장치 변경 감지 및 프로파일 선택 상태 동기화, 실제 장치 기반 프로파일 선택 검증

### Modified Capabilities

## Impact

- 오디오 장치 변경 이벤트 리스너(Audio Endpoint Volume/Device Listener) 추가 또는 수정
- 메인 UI 상태(프로파일 카드 체크 마크 등) 및 트레이 아이콘 업데이트 로직 변경
