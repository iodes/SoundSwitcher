## 1. 백엔드 오디오 이벤트 리스너 구현

- [x] 1.1 백엔드(오디오 관리자)에서 Windows Core Audio API `OnDefaultDeviceChanged` 이벤트 핸들러 구현
- [x] 1.2 시스템 기본 오디오(출력/입력) 장치가 변경되었을 때 이를 감지하는 로직 추가
- [x] 1.3 장치가 변경되었을 때 프론트엔드로 변경 이벤트(변경된 장치 ID 포함)를 전달하는 IPC 채널 구현

## 2. 프론트엔드 상태 관리 및 UI 연동

- [x] 2.1 프론트엔드 전역 상태(Context/Store)에 현재 시스템의 활성 오디오 장치(출력 및 입력) ID 상태 추가
- [x] 2.2 백엔드로부터 전송되는 장치 변경 IPC 이벤트를 수신하여 프론트엔드의 현재 장치 상태 업데이트
- [x] 2.3 앱 초기 구동 시 백엔드에 현재 시스템 장치 상태를 요청하여 동기화하는 로직 추가

## 3. 프로파일 선택 검증 로직 변경

- [x] 3.1 기존 단순히 클릭된 `selectedProfileId` 기반의 프로파일 활성화 판별 로직을 보완(단일 의존성에서 보조 식별자로 역할 변경)
- [x] 3.2 `profile.outputDeviceId === currentSystemOutputId && profile.inputDeviceId === currentSystemInputId` 조건을 필수로 만족하면서, 일치하는 프로파일이 여러 개일 경우 `lastSelectedProfileId`를 통해 최종 선택 프로파일을 결정하는 Selector 로직 구현
- [x] 3.3 프로파일 카드 렌더링 시 새 활성 조건에 맞게 초록색 체크마크 표시되도록 컴포넌트 수정
- [x] 3.4 트레이 아이콘 등 메인 UI 외 다른 영역의 프로파일 선택 상태 표시도 동일한 조건으로 연동되게 수정
- [x] 3.5 프로파일의 콤보박스에서 출력/입력 장치를 변경할 때 즉각 활성화 조건을 재평가하여 트레이 아이콘에 실시간 반영되도록 이벤트 처리 추가

## 4. 테스트 및 검증

- [x] 4.1 UI 내에서 특정 프로파일 클릭 시 시스템 장치가 정상 변경되고, 바뀐 상태를 이벤트로 감지하여 초록 체크가 동기화되는지 확인
- [x] 4.2 Windows 자체 소리 설정 창에서 오디오 출력/입력 장치를 변경했을 때 SoundSwitcher UI의 프로파일 선택 상태가 실시간으로 자동 변경되는지 확인
