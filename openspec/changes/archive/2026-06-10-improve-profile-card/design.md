## Context

현재 프로파일 관리는 단순히 이름과 장치 매핑으로만 제공되며 시각적 구분이 부족합니다. 사용자가 쉽게 자신의 프로파일을 식별할 수 있도록 커스텀 아이콘 또는 시스템 기본 아이콘 표시 기능이 필요합니다. 또한 마우스 오버 시 표시되는 삭제 아이콘이 오작동을 유발할 수 있어 이를 우클릭 컨텍스트 메뉴로 이동하고, 어떤 카드가 조작 중인지 명확히 보여주는 UI(선택 및 포커스) 개선이 필요합니다.

## Goals / Non-Goals

**Goals:**
- 프로파일 상태 데이터에 커스텀 아이콘 경로 속성 추가 (캐시된 경로 사용)
- 아이콘 클릭을 통한 이미지 선택(ICO, PNG, JPEG) 및 앱 데이터 폴더로의 캐싱
- 커스텀 아이콘이 없을 경우 프로파일에 설정된 재생 장치(Playback Device)의 시스템 아이콘을 기본으로 표시
- 우클릭 컨텍스트 메뉴를 통한 프로파일 삭제 및 커스텀 아이콘 초기화 기능 구현
- 삭제 버튼이 차지하던 우측 공간을 장치 선택 콤보박스 등 프로파일 내부 영역이 가로로 꽉 채우도록 레이아웃 확장
- 현재 활성화된 프로파일 아이콘의 우측 하단에 초록색 원/흰색 체크 마크 오버레이 표시
- 전역 설정 항목 "트레이에 프로파일 아이콘 표시" 추가 (기본값: true)
- 옵션 활성화 시 프로파일 아이콘 상태를 시스템 트레이에 동기화, 비활성화 혹은 프로파일 부재 시 SoundSwitcher 앱 로고 표시
- 컨텍스트 메뉴 활성화 시 대상 카드의 스타일 변경 (약한 파란색 포커스)
- 한 번에 하나의 카드만 포커스(선택)되도록 다중 선택 제한

**Non-Goals:**
- 프로파일 카드 외 다른 컴포넌트의 대규모 리팩토링
- 커스텀 아이콘 프리셋 목록 제공 기능

## Decisions

- **컨텍스트 메뉴 및 포커스 상태**: 컴포넌트 상태 혹은 상위 상태로 `focusedProfileId`를 관리. 카드에서 `onContextMenu` 이벤트 발생 시 기본 메뉴를 차단(`e.preventDefault()`)하고, 커스텀 컨텍스트 메뉴를 렌더링함과 동시에 해당 카드를 포커스 상태로 설정.
- **아이콘 데이터 모델 및 캐싱**: 사용자가 ICO, PNG, JPEG 이미지를 선택하면, 백엔드 로직을 통해 앱 데이터 폴더(예: 설정 파일이 있는 경로 하위의 `icons/` 디렉토리)로 복사 및 캐싱합니다. 프로파일 데이터에는 절대 경로가 아닌 캐시된 파일의 상대 경로 또는 식별자(`iconPath`)를 저장하여 원본 파일의 이동이나 삭제에 영향받지 않도록 합니다. 또한, 프로파일 삭제나 아이콘 초기화 시 연결된 캐시 이미지 파일을 로컬 디스크에서 물리적으로 삭제하여 용량 낭비를 방지합니다.
- **아이콘 및 상태 표시 렌더링**: 캐시된 `iconPath` 존재 여부에 따라 이미지를 렌더링. `iconPath`가 없을 경우 프로파일의 재생 장치 ID를 이용해 Windows 시스템의 해당 재생 장치 아이콘을 가져와 Fallback으로 표시합니다. 컨텍스트 메뉴의 '기본 아이콘으로 초기화' 항목을 통해 `iconPath`를 null로 초기화. 전역 상태의 활성 프로파일 ID와 해당 카드의 ID가 일치할 경우, 아이콘 이미지 컨테이너(`relative`) 내의 우측 하단에 체크 마크 SVG 컴포넌트를 `absolute`로 배치.
- **트레이 동기화 및 전역 설정**: 앱 전역 설정에 `showProfileIconInTray` 상태를 추가. 이 값이 `true`이고 활성 프로파일이 존재할 때는 활성 프로파일의 `iconPath`(또는 재생 장치 아이콘)를 트레이에 반영합니다. 값이 `false`이거나 저장된 프로파일이 없을 경우, SoundSwitcher 기본 앱 로고를 트레이 아이콘으로 복구합니다. 이 로직은 아이콘 변경, 프로파일 전환, 설정 토글 시 프론트엔드에서 백엔드 API를 호출하여 즉시 처리됩니다.
- **다중 선택 방지**: 문서(document) 영역에 `mousedown` 이벤트를 리스닝하여, 컨텍스트 메뉴 외부 영역 클릭 시 `focusedProfileId` 상태를 해제하여 다중 선택이 불가능하도록 처리.

## Risks / Trade-offs

- **이미지 로드 에러 리스크**: 캐시된 이미지를 로드하는 과정에서 파일 손상 등으로 문제가 생길 경우 엑스박스가 뜰 수 있음. -> 이미지 `onError` 핸들러를 통해 로드 실패 시 기본 아이콘으로 Fallback 처리.
- **컨텍스트 메뉴 접근성**: 우클릭 외에도 키보드 접근성(Shift+F10 등)을 제공할 수 있으면 좋지만 초기 목표에선 마우스 이벤트 중심으로 우선 구현.
