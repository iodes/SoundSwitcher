## Context

현재 `SoundSwitcher`의 프로파일 카드 내부에는 오디오 장치를 선택할 수 있는 콤보박스(ComboBox)가 제공됩니다. 하지만 장치 목록이 특정 기준(예: 이름순)으로 정렬되어 있지 않아 사용자가 원하는 장치를 찾기 불편하며, 장치 이름이 긴 경우 텍스트가 잘려서 표시되어(Truncation 문제) 전체 이름을 식별하기 어렵습니다. 또한, 콤보박스의 드롭다운을 열었을 때 각 항목이 텍스트로만 표시되어 시각적 구분이 명확하지 않습니다. 이러한 문제를 해결하여 사용자 경험(UX)을 향상시키고자 합니다.

## Goals / Non-Goals

**Goals:**
- 장치 콤보박스 내의 항목들을 알파벳순(ABC순, 오름차순)으로 정렬
- 선택된 장치 이름이 길 경우 '...'으로 말줄임(TextTrimming) 처리되도록 UI 수정
- 콤보박스 드롭다운 목록 내의 개별 항목에 장치 종류를 식별할 수 있는 아이콘(시스템 아이콘 또는 폰트 아이콘 등) 표시 추가

**Non-Goals:**
- 오디오 장치를 수집하는 핵심 비즈니스 로직(Backend API 호출 등) 변경
- 프로파일 카드 자체의 레이아웃 구조 전면 개편

## Decisions

- **목록 정렬 방식**: ViewModel 또는 뷰에서 장치 목록을 렌더링하기 전에 컬렉션을 정렬합니다. 
  - *Rationale*: UI 레이어(또는 프레젠테이션 레이어)에서 `CollectionView` 또는 LINQ의 `OrderBy`를 활용하여 정렬하는 것이 기존 로직에 영향을 덜 미치며 구현이 간단합니다.
- **말줄임(TextTrimming) 적용**: XAML 내 콤보박스의 `ItemTemplate` 또는 선택된 항목 표시 영역 템플릿에서 `TextBlock`의 `TextTrimming="CharacterEllipsis"` 속성을 적용합니다.
  - *Rationale*: 별도의 C# 코드 작성 없이 선언적으로 해결할 수 있어 유지보수가 용이합니다.
- **아이콘 표시 (드롭다운 목록)**: 콤보박스의 `ItemTemplate` 내부 레이아웃을 `StackPanel` 또는 `Grid`로 구성하여 아이콘(`Image` 또는 `FontIcon`)과 텍스트(`TextBlock`)를 나란히 배치합니다. 
  - *Rationale*: 장치 유형(스피커, 마이크, 헤드셋 등)에 맞는 아이콘을 직관적으로 보여주면 사용자가 장치를 훨씬 더 빠르게 식별할 수 있습니다. 이미 다른 곳에서 장치 아이콘을 가져오는 로직이 있다면 이를 재사용합니다.

## Risks / Trade-offs

- [Risk: 긴 장치 이름 식별 불가] 말줄임 처리로 인해 매우 유사하고 긴 장치 이름(끝부분만 다른 경우)을 식별하기 어려울 수 있습니다.
  → Mitigation: 말줄임된 텍스트에 마우스를 올렸을 때 전체 이름을 보여주는 툴팁(`ToolTip`)을 추가하여 보완할 수 있습니다.
- [Risk: 성능 저하] 장치가 매우 많을 때 드롭다운 목록에서 렌더링되는 아이콘 때문에 약간의 로딩 지연이 발생할 수 있습니다.
  → Mitigation: 아이콘 로딩이 비동기적으로 처리되거나 이미 캐싱된 아이콘을 재사용하도록 구현하여 성능 저하를 방지합니다.

## Migration Plan

기존 코드를 기반으로 XAML 파일 및 관련 ViewModel의 정렬 코드만 수정하므로 특별한 데이터 마이그레이션이 필요하지 않습니다.
