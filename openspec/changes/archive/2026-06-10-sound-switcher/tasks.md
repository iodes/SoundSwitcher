## 1. 프로젝트 초기 설정

- [x] 1.1 새 WPF 프로젝트(SoundSwitcher) 생성 (.NET 10 기반)
- [x] 1.2 필요한 NuGet 패키지 설치 (WPF-UI, NAudio 또는 CoreAudioApi 구현체, Hardcodet.NotifyIcon.Wpf)
- [x] 1.3 기본 앱 진입점 수정하여 시작 시 숨김 상태(트레이 전용)로 실행되도록 구성

## 2. 오디오 장치 제어 및 데이터 모델 기능 구현

- [x] 2.1 시스템의 활성 오디오 장치 목록을 출력(Playback)과 입력(Capture)으로 구분하여 조회하는 기능 구현
- [x] 2.2 시스템의 현재 기본 장치 및 기본 통신 장치를 가져오는 기능 구현
- [x] 2.3 특정 오디오 장치를 시스템 기본 장치 및 통신 장치로 변경하는 기능 구현
- [x] 2.4 사용자 설정 데이터 모델(선호 장치 ID 목록, 커스텀 아이콘 경로, 순서 인덱스, 통신 장치 동시 전환 여부, 시작프로그램 등록 여부) 생성
- [x] 2.5 로컬 JSON 파일 기반의 설정 불러오기 및 저장 로직 구현

## 3. UI 구현: 장치 / 설정 / 정보 탭 구성

- [x] 3.1 메인 윈도우(설정 창)에 WPF-UI의 NavigationView 등을 활용하여 3개의 탭(장치, 설정, 정보) 레이아웃 구성
- [x] 3.2 **[장치 탭]** 출력 장치와 입력 장치를 구분하여 보여주는 리스트 구현
- [x] 3.3 **[장치 탭]** 각 장치 항목에 선호 장치 선택용 체크박스 및 아이콘 변경 버튼 구현
- [x] 3.4 **[장치 탭]** 체크된 장치들의 순서를 드래그 앤 드롭으로 재배치하는 로직 구현
- [x] 3.5 **[설정 탭]** '기본 통신 장치 동시 전환' 체크박스 토글 구현 및 설정 데이터 바인딩
- [x] 3.6 **[설정 탭]** 'Windows 시작 시 자동 실행' 체크박스 토글 구현 (레지스트리 또는 시작프로그램 폴더 연동)
- [x] 3.7 **[정보 탭]** 앱 버전 번호 및 개발자 정보 표시 UI 구현

## 4. 시스템 트레이 및 전환 핵심 로직 통합

- [x] 4.1 현재 기본 장치 상태와 선호 장치 목록의 커스텀 순서를 바탕으로 '다음 전환할 장치'를 결정하는 순환 로직 구현
- [x] 4.2 시스템 트레이 아이콘 표시 및 설정된 장치 아이콘(없을 시 Windows 기본 아이콘) 로드 로직 구현
- [x] 4.3 트레이 아이콘 좌클릭 이벤트에 장치 전환 기능 연결 및 전환 후 트레이 아이콘/툴팁 동적 갱신
- [x] 4.4 트레이 아이콘 우클릭 컨텍스트 메뉴(설정 창 띄우기, 종료) 구현
- [x] 4.5 설정 창에서의 변경 사항(아이콘, 순서, 통신 장치 옵션 등)이 즉각적으로 트레이 동작에 반영되도록 이벤트 연동
