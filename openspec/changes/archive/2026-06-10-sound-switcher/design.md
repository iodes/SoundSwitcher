## Context

Windows에서 기본 제공하는 오디오 장치 변경 방법은 시스템 트레이의 볼륨 아이콘을 클릭하고, 확장 메뉴를 열어 장치를 선택하는 과정을 거칩니다. 헤드셋과 스피커를 자주 번갈아 사용하는 사용자에게는 이러한 반복적인 과정이 번거롭게 느껴집니다.

## Goals / Non-Goals

**Goals:**
- 시스템 트레이 아이콘을 한 번 클릭하는 것만으로 사용자가 미리 지정한 오디오 장치들 사이를 순환하며 기본 장치로 설정.
- 사용자 친화적인 3개의 탭(장치, 설정, 정보) 기반 설정 화면 제공.
- 출력 및 입력 장치를 구분하여 관리.
- 장치 순서 재배치 및 개별 장치 아이콘 커스터마이징.
- 최소한의 리소스만 사용하는 가벼운 백그라운드 프로세스로 동작.

**Non-Goals:**
- 개별 애플리케이션 볼륨 조절 기능 (Windows 볼륨 믹서 기능 대체 아님).
- 볼륨 단계를 조절하는 기능.
- 고급 오디오 이퀄라이저 설정.

## Decisions

- **Framework / UI**: .NET 10 WPF 및 WPF-UI (https://github.com/lepoco/wpfui). (이유: UI가 필요한 설정 창 구현이 용이하며, WPF-UI를 통해 Windows 11 스타일의 모던하고 깔끔한 체크박스 기반 리스트 UI를 쉽게 구현할 수 있음.)
- **UI Structure**: 
  - **장치 탭**: 출력/입력 카테고리를 분리. 체크박스로 선호 장치 선택. 드래그 앤 드롭으로 순서 변경. 장치별 아이콘 변경 기능.
  - **설정 탭**: 전환 시 통신 장치 동시 전환 여부, Windows 시작 시 자동 실행 여부.
  - **정보 탭**: 앱 버전 및 개발자 정보.
- **Audio API**: NAudio / AudioSwitcher.AudioApi (또는 Core Audio API 래퍼). (이유: 재생/녹음 장치 목록을 불러오고 기본 및 통신 장치를 설정하는 데 필요.)
- **Tray Icon Library**: Hardcodet.NotifyIcon.Wpf. (이유: WPF에서 네이티브 트레이 아이콘을 매우 쉽게 구현하고, 마우스 클릭 이벤트 및 동적 아이콘(유저 설정 아이콘) 업데이트를 원활하게 처리할 수 있음.)
- **Storage**: 로컬 JSON 파일. (이유: 사용자 설정 데이터(순서, 아이콘 경로, 설정값)가 적어 복잡한 데이터베이스 구조가 불필요함.)

## Risks / Trade-offs

- [Risk] 장치별 개별 커스텀 아이콘 리소스 관리. → [Mitigation] 유저가 지정한 아이콘 파일을 앱 데이터 폴더로 복사하거나 경로를 안전하게 참조. 설정된 아이콘이 없거나 삭제된 경우 기본 Windows 장치 아이콘으로 Fallback하는 로직 구현.
