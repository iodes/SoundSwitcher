## Context

최근 트레이 컨텍스트 메뉴의 애니메이션 효과를 수정하는 과정에서, 메뉴 외곽에 적용되던 그림자(Drop Shadow) 효과가 사라지는 문제가 발생했습니다. WPF에서는 보통 `Popup` 내부의 `Border`에 `DropShadowEffect`를 주거나 시스템 테마(SystemDropShadowChrome 등)를 사용하여 그림자를 렌더링하는데, 팝업 애니메이션을 위한 레이아웃 변경 과정에서 그림자가 잘리거나 효과가 제거되었을 수 있습니다.

## Goals / Non-Goals

**Goals:**
- 트레이 컨텍스트 메뉴의 테두리 그림자 효과 복구
- 기존에 적용된 메뉴 열기/닫기 애니메이션이 그림자 효과와 충돌 없이 정상 동작하도록 유지
- 다른 WPF 기본 컨텍스트 메뉴와 시각적 일관성 확보

**Non-Goals:**
- 트레이 메뉴 외의 다른 UI 컴포넌트 스타일 변경
- 메뉴 애니메이션 로직 자체의 대대적인 재작성

## Decisions

- **DropShadowEffect 적용 및 여백 확보**:
  - `TrayIconView.xaml` (또는 ContextMenu 스타일이 정의된 리소스)의 최상위 `Border`에 `DropShadowEffect`를 다시 선언합니다.
  - 그림자가 렌더링될 공간이 부족하여 잘리는(Clip) 현상을 방지하기 위해, 애니메이션을 감싸는 컨테이너 또는 `Border`에 적절한 `Margin`을 부여하여 그림자가 표시될 여백을 확보합니다.

## Risks / Trade-offs

- **애니메이션과 그림자 렌더링의 충돌 위험**: `DropShadowEffect`가 적용된 엘리먼트에 애니메이션을 적용하면 렌더링 성능이 저하되거나 시각적 글리치가 발생할 수 있습니다.
  - **Mitigation**: 하드웨어 가속 성능을 고려하여 렌더링 트리를 최적화하고, 필요 시 그림자 요소와 애니메이션 요소를 분리하는 방안을 고려합니다.
