## MODIFIED Requirements

### Requirement: 즉각적인 장치 전환 알림 제공
시스템은 장치 전환이 발생했을 때 전환된 장치 정보를 담은 시각적 알림(Notification)을 화면에 표시해야 한다. 이 알림은 OS의 기본 큐(Queue)에 의존하지 않고 즉각적으로 나타나야 하며, 사용자가 연속으로 장치를 전환할 경우 알림이 중첩되거나 지연되지 않도록 이전 알림을 즉시 덮어쓰거나 닫아야 한다. 또한, 이전 알림이 닫히는 애니메이션 도중 새로운 장치 전환이 발생하더라도 닫힘 애니메이션을 올바르게 취소하고 즉각적으로 새 알림을 온전히 표시하여 알림창이 투명한 상태로 멈추거나 닫힘 기능이 오작동하는 일이 없어야 한다.

#### Scenario: 단일 장치 전환 시 알림 발생
- **WHEN** 사용자가 트레이 아이콘을 클릭하여 새로운 프로파일로 장치를 전환할 때
- **THEN** 시스템은 딜레이 없이 전환된 장치 이름과 아이콘을 포함한 알림 창을 즉시 표시해야 한다

#### Scenario: 연속적인 장치 전환 시 알림 중복 방지
- **WHEN** 기존 알림 창이 화면에 표시된 상태에서 사용자가 다시 트레이 아이콘을 클릭하여 다른 프로파일로 장치를 연속 전환할 때
- **THEN** 시스템은 기존에 떠 있던 알림 창을 즉시 닫고, 새롭게 전환된 장치 정보가 담긴 알림 창으로 대체하여 화면에 알림이 여러 개 쌓이지 않도록 해야 한다

#### Scenario: 알림 닫힘 애니메이션 도중 전환 시 표시 오류 방지
- **WHEN** 알림 창이 닫히는(페이드아웃 등) 애니메이션이 실행되는 짧은 타이밍에 사용자가 장치를 연속 전환할 때
- **THEN** 시스템은 진행 중이던 닫힘 애니메이션과 닫힘 상태를 즉각 중단 및 초기화하고, 투명도를 원상복구하여 새 장치 정보가 정상적으로 보이도록 업데이트해야 한다
