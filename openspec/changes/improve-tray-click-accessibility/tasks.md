## 1. 트레이 아이콘 클릭 동작 수정

- [ ] 1.1 시스템 트레이 아이콘의 좌클릭 이벤트 핸들러(예: `App.xaml.cs`의 `TaskbarIcon_TrayLeftMouseDown`)를 찾습니다.
- [ ] 1.2 해당 핸들러 내부에서 `ProfileManager.Profiles.Count` 값을 확인하는 조건을 추가합니다.
- [ ] 1.3 프로파일 개수가 0개일 때, 메인 창을 여는 기존 로직(예: `ShowMainWindowCommand.Execute(null)` 또는 `MainWindow.Show()`)을 호출하도록 구현합니다.
- [ ] 1.4 프로파일 개수가 1개일 때, 불필요한 프로파일 전환 시도를 방지하기 위해 로직을 즉시 종료(return)하도록 방어 코드를 추가합니다.
- [ ] 1.5 프로파일 개수가 2개 이상일 때는 기존의 프로파일 전환 로직이 정상적으로 실행되는지 확인합니다.

## 2. 검증 단계

- [ ] 2.1 프로파일이 0개일 때 트레이 아이콘을 좌클릭하면 메인 창이 열리는지 확인합니다.
- [ ] 2.2 프로파일이 1개일 때 트레이 아이콘을 좌클릭하면 아무 동작도 하지 않고 예외가 발생하지 않는지 확인합니다.
- [ ] 2.3 프로파일이 2개 이상일 때 트레이 아이콘을 좌클릭하면 정상적으로 다음 장치로 전환되는지 확인합니다.
