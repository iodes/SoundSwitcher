## 2. ProfileCard 컴포넌트 마크업 및 스타일링 업데이트

- [x] 2.1 기존 마우스 오버 시 표시되던 카드 우측 삭제 버튼 마크업 및 공간을 제거하고, 장치 선택 콤보박스 등 카드 내부 영역이 남은 가로 폭을 모두 차지하도록(`flex-grow` 등) 레이아웃 수정
- [x] 2.2 드래그 재정렬 그립 다음에 아이콘을 표시할 영역 컨테이너(`relative`) 추가
- [x] 2.3 `iconPath` 데이터가 있을 경우 해당 이미지 표시, 없을 경우 프로파일에 설정된 **재생 장치(Playback Device)**의 아이콘을 백엔드로부터 가져와 렌더링
- [x] 2.4 현재 활성화된 프로파일인지 판별하여, 활성 상태인 경우 아이콘 우측 하단에 초록색 원 모양의 흰색 체크 마크 SVG를 `absolute`로 렌더링
- [x] 2.5 카드의 `focusedProfileId` 일치 여부에 따라 백그라운드 색상을 약한 파란색으로 변경하는 조건부 스타일링 추가

- [x] 3.3 지정된 `iconPath`에 해당하는 캐시 파일을 디스크에서 삭제하는 백엔드 API/로직 구현
- [x] 3.4 반환된 캐시 경로를 프론트엔드 상태의 `iconPath`에 저장하고 렌더링에 사용하도록 연결
- [x] 3.5 카드 영역 전체에 `onContextMenu` 핸들러를 추가하여 기본 메뉴 차단 및 `focusedProfileId` 설정

- [x] 3.8 프로파일 변경, 아이콘 변경, 설정 체크 토글 발생 시 동작하는 트레이 아이콘 업데이트 백엔드 로직 작성 (`showProfileIconInTray`가 false이거나 활성 프로파일이 없을 경우 SoundSwitcher 기본 로고 적용)
