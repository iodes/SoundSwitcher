## Why

현재 프로파일 카드 UI에서는 각 프로파일의 아이콘을 직관적으로 확인하거나 수정하기 어렵고, 마우스 오버 시 표시되는 삭제 버튼은 의도치 않은 클릭을 유발할 수 있습니다. 또한 다중 선택이 불가능한 상태에서 카드 선택에 대한 시각적인 피드백을 더 명확하게 제공하여 사용자 경험을 개선할 필요가 있습니다.

## What Changes

- **프로파일 아이콘 표시**: 드래그 재정렬 그립 다음에 아이콘 표시 영역을 추가합니다. 기본값으로 해당 프로파일에 설정된 재생 장치(Playback Device)의 Windows 기본 아이콘이 표시됩니다.
- **아이콘 변경 및 초기화**: 아이콘을 클릭하여 다른 이미지(ICO, PNG, JPEG 지원)로 변경할 수 있도록 합니다. 선택된 이미지는 원본의 이동이나 삭제에 영향받지 않도록 앱 데이터 폴더에 캐싱되어 관리됩니다. 우클릭 컨텍스트 메뉴에 '기본 아이콘으로 초기화' 옵션을 제공하여 기본 아이콘으로 초기화할 수 있는 기능을 추가합니다.
- **트레이 아이콘 동기화 및 설정**: 현재 선택된(활성화된) 프로파일의 아이콘(커스텀 또는 기본 재생 장치 아이콘)이 시스템 트레이 아이콘에도 즉시 반영되도록 동기화합니다. 이를 제어하기 위해 전역 설정에 "트레이에 프로파일 아이콘 표시" 옵션(기본값: 활성화)을 추가하며, 이 옵션이 꺼져 있거나 등록된 프로파일이 하나도 없을 경우에는 SoundSwitcher 앱 기본 로고가 트레이에 표시됩니다.
- **활성 프로파일 시각적 피드백**: 현재 선택된(활성화된) 프로파일의 경우, 아이콘 우측 하단에 초록색 원 안에 흰색 체크 표시를 오버레이하여 직관적으로 상태를 알 수 있게 합니다.
- **컨텍스트 메뉴를 통한 관리 (BREAKING)**: 카드에 마우스를 올렸을 때 보이던 기존의 삭제 버튼을 제거하고, 남은 공간을 장치 선택 콤보박스 등 카드 내부 영역이 가로로 꽉 채우도록 합니다. 또한 카드 우클릭 시 나타나는 컨텍스트 메뉴를 통해 프로파일 삭제 및 커스텀 아이콘 초기화 기능을 제공하도록 변경합니다.
- **선택 및 포커스 시각화**: 우클릭으로 컨텍스트 메뉴가 열린 카드는 약한 파란색으로 포커스 처리하여 어떤 프로파일을 조작 중인지 명확히 표시합니다. 다중 선택은 불가능하도록 제한합니다.

## Capabilities

### New Capabilities
- `profile-management`: 프로파일 카드의 아이콘 커스터마이징(변경 및 초기화) 및 컨텍스트 메뉴를 통한 삭제/초기화와 포커스 상태 관리에 대한 요구사항 명세

### Modified Capabilities

## Impact

- React UI의 프로파일 카드 컴포넌트 구조 변경 (`ProfileCard` 등)
- 프로파일 데이터 상태 관리에 커스텀 아이콘 속성 추가 필요
- 마우스 우클릭 이벤트 핸들링 및 컨텍스트 메뉴 컴포넌트 도입
- 카드 포커스 시각 효과를 위한 CSS 스타일 업데이트
