# Capability: profile-state-sync

## Purpose
프로파일과 실제 시스템 장치 상태를 동기화하고, 시스템 장치 변경 이벤트를 감지하여 UI를 업데이트합니다.

## Requirements

### Requirement: 실제 시스템 장치 기반 프로파일 선택 검증
시스템은 현재 선택된 프로파일이 무엇인지 판단할 때 단순히 사용자가 마지막으로 클릭한 프로파일 ID를 따르는 대신, 프로파일에 지정된 입력 및 출력 장치 ID가 현재 시스템에 실제로 선택(활성화)되어 있는 기본 장치와 일치하는 경우에만 해당 프로파일이 "선택"된 것으로 판단해야 한다.
단, 동일한 입출력 장치 구성을 가진 프로파일이 여러 개 존재할 수 있으므로, 장치가 일치하는 프로파일들 중에서는 사용자가 마지막으로 선택했던 프로파일 ID(`lastSelectedProfileId`)를 가진 프로파일을 최종 선택된 것으로 식별한다. `lastSelectedProfileId`를 가진 프로파일이라도 현재 시스템의 실제 장치와 다르면 무조건 선택 상태가 해제되어야 한다.

#### Scenario: 프로파일과 실제 장치가 일치할 때
- **WHEN** 현재 시스템의 오디오 출력 및 입력 장치가 특정 프로파일(A)의 출력/입력 장치와 모두 일치할 때
- **THEN** 시스템은 프로파일(A)를 현재 선택된 프로파일로 판단하고, 프로파일 카드 목록 및 트레이 등에서 (A) 프로파일에만 초록색 체크 표시 등 선택 상태 피드백을 제공해야 한다

#### Scenario: 동일 장치를 가진 프로파일이 여러 개일 때
- **WHEN** 현재 시스템의 장치와 일치하는 프로파일이 (A), (B) 두 개가 존재하고, 사용자가 이전에 (B)를 명시적으로 클릭(선택)하여 마지막 선택 ID가 (B)인 상태일 때
- **THEN** (A)와 (B) 모두 실제 장치 조건은 만족하지만, 마지막 선택 ID와 일치하는 프로파일(B)에만 초록색 체크 표시 등 선택 상태 피드백을 제공해야 한다

#### Scenario: 외부 요인으로 시스템 장치가 변경되어 불일치할 때
- **WHEN** 프로파일(A)가 마지막으로 선택되어 활성화된 상태에서, 사용자가 Windows 설정 등 외부에서 시스템 오디오 출력 장치를 다른 장치로 변경할 때
- **THEN** 시스템의 현재 활성 장치와 프로파일(A)의 지정 장치가 더 이상 일치하지 않으므로, 마지막 선택 ID가 (A)이더라도 프로파일(A)의 선택 상태(초록색 체크 표시)는 즉시 해제되어야 한다

### Requirement: 오디오 장치 변경 이벤트 감지 및 UI 동기화
시스템은 OS 차원의 오디오 장치 기본값 변경 이벤트(출력 또는 입력 장치 변경)를 실시간으로 감지하여, 내부 선택 상태와 UI(프로파일 목록 등)를 갱신해야 한다.

#### Scenario: 외부 프로그램에서 장치 변경
- **WHEN** 다른 애플리케이션이나 OS 설정에서 기본 오디오 장치를 변경하여 시스템 장치가 프로파일(B)의 장치와 일치하게 될 때
- **THEN** 시스템은 장치 변경 이벤트를 감지하고 UI를 갱신하여, 사용자 개입 없이 즉시 프로파일(B)가 선택된 상태로 변경(초록색 체크 표시)하여 보여주어야 한다

#### Scenario: 콤보박스 변경에 따른 트레이 아이콘 및 상태 동기화
- **WHEN** 사용자가 활성화된 프로파일의 출력/입력 장치를 콤보박스에서 다른 장치로 변경할 때
- **THEN** 시스템은 해당 변경 즉시 활성화 조건을 재평가하고, 활성화 프로파일 상태 변경에 맞춰 트레이 아이콘을 즉각적으로 최신 상태(새로운 프로파일 아이콘 또는 기본 아이콘)로 동기화해야 한다
