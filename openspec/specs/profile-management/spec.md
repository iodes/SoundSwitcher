# Capability: profile-management

## Purpose
프로파일 카드 UI 상에서 프로파일별 고유 아이콘 표시 및 설정 기능을 제공하며, 삭제 기능을 우클릭 컨텍스트 메뉴로 통합하여 사용자 조작의 정확성을 높입니다.

## ADDED Requirements

### Requirement: 프로파일 카드 상호작용 시각적 피드백
프로파일 카드는 클릭 가능한 요소임을 사용자에게 명확히 전달하기 위해 마우스 상호작용에 따른 시각적 피드백을 제공해야 한다.

#### Scenario: 마우스 오버 시 시각적 피드백
- **WHEN** 사용자가 마우스 커서를 프로파일 카드 위로 올릴 때
- **THEN** 프로파일 카드의 배경색이 미세하게 밝아지거나 어두워져 마우스 오버 상태임을 나타내야 한다

#### Scenario: 마우스 클릭(다운) 시 시각적 피드백
- **WHEN** 사용자가 프로파일 카드를 마우스로 좌클릭(다운)할 때
- **THEN** 프로파일 카드의 크기가 미세하게 줄어들거나 눌린듯한 시각적 효과가 발생하여 클릭 동작이 인지되어야 한다

### Requirement: 프로파일 카드 아이콘 표시 및 설정
프로파일 카드는 드래그 재정렬 그립 옆에 아이콘을 표시해야 한다. 유저가 별도로 커스텀 아이콘을 지정하지 않으면 해당 프로파일에 설정된 **재생 장치(Playback Device)**의 시스템 아이콘을 기본값으로 사용해야 한다. 사용자는 컨텍스트 메뉴를 통해 커스텀 이미지(ICO, PNG, JPEG)로 아이콘을 변경할 수 있어야 한다. 설정된 커스텀 이미지는 원본 파일 위치에 상관없이 유지되도록 앱 데이터 폴더 내에 캐싱되어야 한다.

#### Scenario: 기본 재생 장치 아이콘 표시
- **WHEN** 사용자가 커스텀 아이콘을 설정하지 않았거나 '기본 아이콘으로 초기화'를 수행한 상태일 때
- **THEN** 프로파일 카드 내에 해당 프로파일에 매핑된 재생 장치의 Windows 시스템 아이콘이 자동으로 렌더링되어야 한다

#### Scenario: 커스텀 아이콘 선택 및 캐싱
- **WHEN** 사용자가 해당 카드의 우클릭 컨텍스트 메뉴에서 '아이콘 변경' 항목을 좌클릭하여 지원되는 이미지 파일(ICO, PNG, JPEG)을 선택할 때
- **THEN** 시스템은 선택된 이미지를 앱 데이터 폴더로 복사/캐싱해야 한다
- **AND** 설정에는 캐시된 이미지의 절대 경로가 아닌, 파일 이름(또는 캐시 폴더 기준 상대 경로)만 저장하여 데이터 이식성을 보장해야 한다
- **AND** UI에서 아이콘을 렌더링할 때 저장된 파일 이름을 기반으로 실제 절대 경로를 동적으로 해석해야 한다

#### Scenario: 저장된 아이콘 캐시 파일 유실 혹은 경로 손상 시의 폴백
- **WHEN** 설정에 커스텀 아이콘 경로(파일 이름)가 저장되어 있으나, 해당 파일이 실제 캐시 디렉터리에 존재하지 않거나 손상된 경우
- **THEN** 시스템은 이를 아이콘 미설정 상태와 동일하게 취급하여 기본 재생 장치 아이콘을 렌더링해야 한다
- **AND** 오류가 발생하더라도 앱이 종료되거나 프로파일 로딩이 실패하지 않아야 한다

### Requirement: 아이콘 초기화
커스텀 아이콘이 설정된 프로파일 카드는 우클릭 시 나타나는 컨텍스트 메뉴에 '기본 아이콘으로 초기화' 항목을 표시해야 하며, 클릭 시 아이콘이 기본값으로 초기화되어야 한다. 초기화 시 시스템에 저장되었던 캐시 이미지 파일도 함께 삭제되어야 한다.

#### Scenario: 커스텀 아이콘 초기화 및 캐시 삭제
- **WHEN** 커스텀 아이콘이 적용된 프로파일에서 사용자가 컨텍스트 메뉴의 '기본 아이콘으로 초기화' 항목을 좌클릭할 때
- **THEN** 해당 카드의 아이콘이 다시 기본 재생 장치 아이콘으로 복구되며, 해당 프로파일과 연결되었던 앱 데이터 폴더 내의 캐시 이미지 파일이 물리적으로 삭제되어야 한다

### Requirement: 트레이 아이콘 동기화 및 설정 제어
현재 활성화된 프로파일의 아이콘 변경 사항은 시스템 트레이 아이콘에 반영될 수 있어야 하며, 이는 전역 설정인 '트레이에 프로파일 아이콘 표시' 옵션에 의해 제어된다.

#### Scenario: 활성 프로파일의 아이콘 트레이 동기화
- **WHEN** '트레이에 프로파일 아이콘 표시' 설정이 켜져 있는 상태에서, 활성화된 프로파일이 변경되거나 해당 프로파일의 아이콘이 업데이트될 때
- **THEN** 대상 아이콘(커스텀 아이콘 혹은 재생 장치 아이콘)이 Windows 시스템 트레이 앱 아이콘으로 즉시 적용되어야 한다

#### Scenario: 설정 비활성화 또는 프로파일 부재 시 앱 로고 표시
- **WHEN** 사용자가 '트레이에 프로파일 아이콘 표시' 설정을 끄거나, 등록된 프로파일이 하나도 없어 활성 프로파일이 존재하지 않을 때
- **THEN** 트레이 아이콘은 SoundSwitcher 앱의 기본 로고 이미지로 표시(또는 복구)되어야 한다

### Requirement: 활성 프로파일 시각적 식별
현재 활성화(선택)되어 있는 프로파일은 사용자가 쉽게 인지할 수 있도록 아이콘 위에 별도의 시각적 표시가 제공되어야 한다. 또한, 프로파일 전환이 대기 중인 상태일 때는 기존 활성 상태 표시 대신 로딩 스피너를 표시하여 사용자에게 전환 대기 중임을 시각적으로 알려야 한다.

#### Scenario: 활성 프로파일 체크 마크 표시
- **WHEN** 시스템의 현재 활성화된 프로파일과 해당 카드가 나타내는 프로파일이 일치할 때
- **THEN** 해당 프로파일 카드의 아이콘 우측 하단에 초록색 원 안에 흰색 체크 모양의 마크가 겹쳐서 표시되어야 한다

#### Scenario: 대기 상태의 프로파일 시각적 표시
- **WHEN** 사용자가 전환을 시도한 프로파일의 장치가 비활성 상태여서 프로파일이 '대기(Pending)' 상태로 전환되었을 때
- **THEN** 해당 프로파일 카드의 아이콘 우측 하단에 초록색 체크 마크 대신 로딩 모양의 스피너(Loading Spinner)가 표시되어 장치가 활성화되기를 기다리고 있음을 나타내야 한다

### Requirement: 컨텍스트 메뉴를 통한 프로파일 관리 기능
프로파일 카드 내의 오동작 방지 및 기능 응집을 위해, 프로파일 관리 및 설정 기능은 카드 전체 영역에 대한 우클릭 컨텍스트 메뉴를 통해 제공되어야 한다. 컨텍스트 메뉴는 논리적인 기능 단위로 그룹화되어야 하며, 각 그룹 사이에는 디바이더가 위치해야 한다. 메뉴의 구조는 다음과 같이 재구성된다:
1. 기본 프로파일 설정 관련 항목 (기본값으로 설정 등)
2. 디바이더
3. 아이콘 관련 메뉴들 (아이콘 변경, 기본 아이콘으로 초기화)
4. 디바이더
5. 출력 장치 속성 (네이티브 속성 창 호출)
6. 입력 장치 속성 (네이티브 속성 창 호출)
7. 디바이더
8. 삭제

사용자가 메뉴 항목을 빠르고 직관적으로 식별할 수 있도록 컨텍스트 메뉴의 각 항목에는 알맞은 기능의 아이콘이 함께 표시되어야 한다. 

#### Scenario: 삭제 및 관리 컨텍스트 메뉴 호출 및 메뉴 구조 그룹화 표시
- **WHEN** 사용자가 프로파일 카드 위에서 우클릭할 때
- **THEN** 프로파일 카드 위치 근처에 관리 옵션이 포함된 컨텍스트 메뉴가 나타나야 한다
- **AND** 컨텍스트 메뉴는 기본 설정, 아이콘, 속성, 삭제 항목들로 그룹화되어 있으며 각 그룹 사이에 디바이더가 표시되어야 한다
- **AND** 컨텍스트 메뉴의 각 항목 왼쪽에는 항목의 의미를 시각적으로 나타내는 아이콘이 표시되어야 한다

#### Scenario: 출력/입력 장치 속성 항목 비활성화 상태
- **WHEN** 프로파일에 출력 장치 또는 입력 장치가 할당되지 않은 상태에서 컨텍스트 메뉴를 열 때
- **THEN** 할당되지 않은 장치에 대응하는 '속성' 항목은 클릭할 수 없도록 비활성화(Disabled) 처리되어야 한다

#### Scenario: 메뉴를 통한 삭제 실행 및 캐시 삭제
- **WHEN** 사용자가 컨텍스트 메뉴의 '삭제' 옵션을 좌클릭할 때
- **THEN** 해당 프로파일이 목록에서 즉시 제거되며, 해당 프로파일이 커스텀 아이콘을 사용 중이었다면 앱 데이터 폴더 내의 연관된 캐시 이미지 파일도 물리적으로 삭제되어야 한다

### Requirement: 프로파일 카드 포커스 시각화
사용자가 어떤 카드를 조작하려는지 명확히 알 수 있도록, 컨텍스트 메뉴가 활성화된 대상 프로파일 카드는 시각적 포커스 효과(약한 파란색 배경)를 가져야 하며 다중 선택은 허용되지 않는다.

#### Scenario: 우클릭에 의한 포커스 처리
- **WHEN** 사용자가 특정 프로파일 카드에 우클릭하여 컨텍스트 메뉴를 활성화할 때
- **THEN** 해당 프로파일 카드의 배경이 약한 파란색으로 변경되어 포커스 상태임을 시각적으로 표시해야 한다

#### Scenario: 포커스 해제 및 다중 선택 제한
- **WHEN** 사용자가 포커스된 카드가 있는 상태에서 컨텍스트 메뉴 외부 영역이나 다른 카드를 클릭(우클릭/좌클릭 모두 포함)할 때
- **THEN** 기존 컨텍스트 메뉴는 닫히고 기존 카드의 포커스 효과가 해제되며, 새로운 상호작용 대상만 포커스 상태를 가져야 한다 (즉, 동시에 두 개 이상의 카드가 약한 파란색 포커스 상태를 가질 수 없다)

### Requirement: 프로파일 카드 클릭을 통한 활성화
사용자는 프로파일 카드의 일반 영역(아이콘 영역 포함, 단 콤보박스 및 드래그 핸들 제외)을 좌클릭하여 해당 프로파일을 활성화(전환)할 수 있어야 하며, 전환 시 사용자 설정에 따라 프로파일 변경 알림이 표시되어야 한다.

#### Scenario: 일반 영역 및 아이콘 영역 좌클릭을 통한 전환 및 알림 표시
- **WHEN** 사용자가 프로파일 카드의 정보가 표시되는 일반 영역 혹은 아이콘 영역을 좌클릭할 때
- **THEN** 해당 카드의 프로파일이 즉시 활성화(선택)되고 사운드 재생 장치가 해당 프로파일에 지정된 장치로 전환되어야 한다
- **AND** '프로파일 변경 알림 표시' 설정이 켜져 있는 경우, 화면에 프로파일 변경 알림이 표시되어야 한다

#### Scenario: 내부 컨트롤과 상호작용 시 전환 방지
- **WHEN** 사용자가 프로파일 카드 내부의 연결 장치 선택 콤보박스, 혹은 좌측 드래그 핸들을 좌클릭할 때
- **THEN** 각각의 컨트롤 고유 기능만 수행되며, 프로파일 전환 이벤트는 발생하지 않아야 한다

### Requirement: 프로파일 변경 알림 표시 설정
사용자는 애플리케이션 설정을 통해 프로파일 변경 시 알림 표시 여부를 제어할 수 있어야 한다. 이 설정은 핫키 및 메인 UI를 통한 모든 프로파일 전환 방법에 공통으로 적용된다.

#### Scenario: 알림 설정 비활성화 상태에서 전환
- **WHEN** 사용자가 '프로파일 변경 알림 표시' 설정을 끄고 핫키나 프로파일 카드를 클릭하여 프로파일을 전환할 때
- **THEN** 프로파일 및 오디오 장치 전환은 정상적으로 수행되지만, 변경 안내 알림은 화면에 표시되지 않아야 한다

### Requirement: 장치 콤보박스 목록 정렬
프로파일 카드 내부의 장치 선택 콤보박스는 드롭다운 목록을 표시할 때 항상 오디오 장치의 이름을 기준으로 알파벳순(ABC순, 오름차순)으로 정렬하여 표시해야 한다.

#### Scenario: 콤보박스 드롭다운 확인 시 정렬
- **WHEN** 사용자가 프로파일 카드 내의 장치 선택 콤보박스를 클릭하여 드롭다운 목록을 열었을 때
- **THEN** 목록 내의 장치들이 모두 알파벳순(ABC순)으로 오름차순 정렬되어 나타나야 한다

### Requirement: 장치 콤보박스 선택 항목 텍스트 말줄임 처리
장치 선택 콤보박스의 선택된 장치 이름이 콤보박스의 가로 너비보다 긴 경우, 텍스트의 끝 부분을 '...'으로 말줄임(Character Ellipsis) 처리하여 UI 레이아웃이 깨지지 않도록 해야 한다.

#### Scenario: 긴 이름의 장치가 선택되었을 때
- **WHEN** 이름이 매우 긴 오디오 장치가 콤보박스에 선택되어 표시될 때
- **THEN** 콤보박스 영역을 벗어나는 텍스트는 잘리고 끝에 '...'이 표시되어야 한다

### Requirement: 장치 콤보박스 목록 내 아이콘 표시
콤보박스의 드롭다운을 펼쳐 장치 목록을 확인할 때, 각 텍스트 항목 좌측에 해당 장치의 시각적 식별을 돕는 장치 아이콘을 함께 표시해야 한다.

#### Scenario: 콤보박스 드롭다운 펼침 시 아이콘 확인
- **WHEN** 사용자가 콤보박스를 클릭하여 드롭다운 목록을 펼쳤을 때
- **THEN** 목록 내의 각 장치 이름 텍스트 옆에 해당 장치 유형(예: 스피커, 마이크, 헤드셋 등)을 나타내는 아이콘이 표시되어야 한다

### Requirement: 프로파일 추가 버튼 상호작용 시각적 피드백
프로파일 추가 버튼은 클릭 가능한 요소임을 명확히 전달하고, 앱 UI에 자연스럽게 녹아들기 위해 부드러운 테두리, 흐림 효과, 마우스 오버 및 클릭 시의 애니메이션(시각적 피드백)을 제공해야 한다.

#### Scenario: 마우스 오버 시 시각적 피드백
- **WHEN** 사용자가 마우스 커서를 프로파일 추가 버튼 위로 올릴 때
- **THEN** 버튼의 배경색이 미세하게 변화하고, 테두리 흐림 효과(또는 둥근 테두리)가 유지되면서 부드러운 전환(Animation)이 발생해야 한다

#### Scenario: 마우스 클릭(다운) 시 시각적 피드백
- **WHEN** 사용자가 프로파일 추가 버튼을 좌클릭(다운)할 때
- **THEN** 버튼의 크기가 미세하게 줄어들거나 눌린 듯한 시각적 애니메이션(Scale Transform 등)이 실행되어 클릭 동작이 확실히 인지되어야 한다

### Requirement: 장치 콤보박스 필터링 및 자동 숨김 동작
프로파일 카드 내부의 장치 선택 콤보박스는 전역 설정인 '사용할 수 없는 장치 표시' 옵션 값에 따라 보여줄 장치 목록을 필터링해야 한다.

#### Scenario: 비활성화 장치 숨김 (옵션 꺼짐)
- **WHEN** '사용할 수 없는 장치 표시' 설정이 비활성화(꺼짐) 상태일 때
- **THEN** 시스템에서 현재 연결이 해제되었거나 사용 불가능한 상태(비활성)인 장치들은 콤보박스 드롭다운 목록에서 제외되어야 한다

#### Scenario: 비활성화 장치 표시 (옵션 켜짐)
- **WHEN** '사용할 수 없는 장치 표시' 설정이 활성화(켜짐) 상태일 때
- **THEN** 연결 해제되거나 비활성화된 장치들도 모두 콤보박스 목록에 포함되어 표시되어야 한다

#### Scenario: 현재 선택된 비활성화 장치의 예외 표시
- **WHEN** '사용할 수 없는 장치 표시' 설정이 비활성화되어 있지만, 특정 프로파일에 이미 비활성 장치가 선택되어 있는 상태일 때
- **THEN** 해당 비활성 장치는 선택 상태를 유지하고 사용자에게 보여지기 위해 예외적으로 콤보박스 목록에 포함되어 표시되어야 한다

#### Scenario: 예외 표시된 장치의 자동 숨김
- **WHEN** 위 예외 조건으로 인해 목록에 표시되던 비활성 장치 대신, 사용자가 콤보박스에서 다른 사용 가능한 장치로 선택을 변경할 때
- **THEN** 해당 비활성 장치는 더 이상 어떠한 프로파일에서도 참조되지 않으므로, 다음 번 장치 목록 갱신 시점부터 콤보박스 목록에서 사라져야(숨김) 한다
