# Capability: profile-management

## Purpose
프로파일 카드 UI 상에서 프로파일별 고유 아이콘 표시 및 설정 기능을 제공하며, 삭제 기능을 우클릭 컨텍스트 메뉴로 통합하여 사용자 조작의 정확성을 높입니다.

## ADDED Requirements

### Requirement: 프로파일 카드 아이콘 표시 및 설정
프로파일 카드는 드래그 재정렬 그립 옆에 아이콘을 표시해야 한다. 유저가 별도로 커스텀 아이콘을 지정하지 않으면 해당 프로파일에 설정된 **재생 장치(Playback Device)**의 시스템 아이콘을 기본값으로 사용해야 한다. 사용자는 아이콘 영역을 클릭하여 커스텀 이미지(ICO, PNG, JPEG)로 아이콘을 변경할 수 있어야 한다. 설정된 커스텀 이미지는 원본 파일 위치에 상관없이 유지되도록 앱 데이터 폴더 내에 캐싱되어야 한다.

#### Scenario: 기본 재생 장치 아이콘 표시
- **WHEN** 사용자가 커스텀 아이콘을 설정하지 않았거나 '기본 아이콘으로 초기화'를 수행한 상태일 때
- **THEN** 프로파일 카드 내에 해당 프로파일에 매핑된 재생 장치의 Windows 시스템 아이콘이 자동으로 렌더링되어야 한다

#### Scenario: 커스텀 아이콘 선택 및 캐싱
- **WHEN** 사용자가 아이콘 영역을 좌클릭하여 지원되는 이미지 파일(ICO, PNG, JPEG)을 선택할 때
- **THEN** 시스템은 선택된 이미지를 앱 데이터 폴더로 복사/캐싱하고, 설정에는 원본의 절대 경로 대신 캐시된 아이콘 경로를 저장하여 아이콘 렌더링에 사용해야 한다

### Requirement: 아이콘 초기화
커스텀 아이콘이 설정된 프로파일 카드는 우클릭 시 나타나는 컨텍스트 메뉴에 '기본 아이콘으로 초기화' 항목을 표시해야 하며, 클릭 시 아이콘이 기본값으로 초기화되어야 한다. 초기화 시 시스템에 저장되었던 캐시 이미지 파일도 함께 삭제되어야 한다.

#### Scenario: 커스텀 아이콘 초기화 및 캐시 삭제
- **WHEN** 커스텀 아이콘이 적용된 프로파일에서 사용자가 컨텍스트 메뉴의 '기본 아이콘으로 초기화' 항목을 좌클릭할 때
- **THEN** 해당 카드의 아이콘이 다시 기본 재생 장치 아이콘으로 복구되며, 해당 프로파일과 연결되었던 앱 데이터 폴더 내의 캐시 이미지 파일이 물리적으로 삭제되어야 한다

### Requirement: 트레이 아이콘 동기화 및 설정 제어
현재 활성화된 프로파일의 아이콘 변경 사항은 시스템 트레이 아이콘에 반영될 수 있어야 하며, 이는 전역 설정인 '트레이에 프로파일 아이콘 표시' 옵션에 의해 제어된다.

#### Scenario: 활성 프로파일의 아이콘 트레이 동기화
- **WHEN** '트레이에 프로파일 아이콘 표시' 설정이 켜져 있는 상태에서, 활성화된 프로파일이 변경되거나 해당 프로파일의 아이콘이 업데이트될 때
- **THEN** 대상 아이콘(커스텀 아이콘 혹은 재생 장치 아이콘)이 Windows 시스템 트레이 앱 아이콘으로 즉시 적용되어야 한다

#### Scenario: 설정 비활성화 또는 프로파일 부재 시 앱 로고 표시
- **WHEN** 사용자가 '트레이에 프로파일 아이콘 표시' 설정을 끄거나, 등록된 프로파일이 하나도 없어 활성 프로파일이 존재하지 않을 때
- **THEN** 트레이 아이콘은 SoundSwitcher 앱의 기본 로고 이미지로 표시(또는 복구)되어야 한다

### Requirement: 활성 프로파일 시각적 식별
현재 활성화(선택)되어 있는 프로파일은 사용자가 쉽게 인지할 수 있도록 아이콘 위에 별도의 시각적 표시가 제공되어야 한다.

#### Scenario: 활성 프로파일 체크 마크 표시
- **WHEN** 시스템의 현재 활성화된 프로파일과 해당 카드가 나타내는 프로파일이 일치할 때
- **THEN** 해당 프로파일 카드의 아이콘 우측 하단에 초록색 원 안에 흰색 체크 모양의 마크가 겹쳐서 표시되어야 한다

### Requirement: 컨텍스트 메뉴를 통한 프로파일 관리 기능
프로파일 카드 내의 오동작 방지를 위해, 삭제 및 아이콘 초기화 기능은 카드 전체 영역에 대한 우클릭 컨텍스트 메뉴를 통해 제공되어야 한다.

#### Scenario: 삭제 컨텍스트 메뉴 호출
- **WHEN** 사용자가 프로파일 카드 위에서 우클릭할 때
- **THEN** 프로파일 카드 위치 근처에 '삭제' 옵션이 포함된 컨텍스트 메뉴가 나타나야 한다

#### Scenario: 메뉴를 통한 삭제 실행 및 캐시 삭제
- **WHEN** 사용자가 컨텍스트 메뉴의 '삭제' 옵션을 좌클릭할 때
- **THEN** 해당 프로파일이 목록에서 즉시 제거되며, 해당 프로파일이 커스텀 아이콘을 사용 중이었다면 앱 데이터 폴더 내의 연관된 캐시 이미지 파일도 물리적으로 삭제되어야 한다

### Requirement: 프로파일 카드 포커스 시각화
사용자가 어떤 카드를 조작하려는지 명확히 알 수 있도록, 컨텍스트 메뉴가 활성화된 대상 프로파일 카드는 시각적 포커스 효과(약한 파란색 배경)를 가져야 하며 다중 선택은 허용되지 않는다.

#### Scenario: 우클릭에 의한 포커스 처리
- **WHEN** 사용자가 특정 프로파일 카드에 우클릭하여 컨텍스트 메뉴를 활성화할 때
- **THEN** 해당 프로파일 카드의 배경이 약한 파란색으로 변경되어 포커스 상태임을 시각적으로 표시해야 한다

#### Scenario: 포커스 해제 및 다중 선택 제한
- **WHEN** 사용자가 포커스된 카드가 있는 상태에서 컨텍스트 메뉴 외부 영역이나 다른 카드를 클릭(우클릭/좌클릭 모두 포함)할 때
- **THEN** 기존 컨텍스트 메뉴는 닫히고 기존 카드의 포커스 효과가 해제되며, 새로운 상호작용 대상만 포커스 상태를 가져야 한다 (즉, 동시에 두 개 이상의 카드가 약한 파란색 포커스 상태를 가질 수 없다)

### Requirement: 프로파일 카드 클릭을 통한 활성화
사용자는 프로파일 카드의 빈 영역(아이콘 영역, 콤보박스, 드래그 핸들 제외)을 좌클릭하여 해당 프로파일을 활성화(전환)할 수 있어야 한다.

#### Scenario: 일반 영역 좌클릭을 통한 전환
- **WHEN** 사용자가 프로파일 카드의 정보가 표시되는 일반 영역(빈 배경 등)을 좌클릭할 때
- **THEN** 해당 카드의 프로파일이 즉시 활성화(선택)되고 사운드 재생 장치가 해당 프로파일에 지정된 장치로 전환되어야 한다

#### Scenario: 내부 컨트롤과 상호작용 시 전환 방지
- **WHEN** 사용자가 프로파일 카드 내부의 아이콘 영역, 연결 장치 선택 콤보박스, 혹은 좌측 드래그 핸들을 좌클릭할 때
- **THEN** 각각의 컨트롤 고유 기능만 수행되며, 프로파일 전환 이벤트는 발생하지 않아야 한다
