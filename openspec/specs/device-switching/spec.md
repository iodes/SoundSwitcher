# Capability: device-switching

## Purpose
TBD: 오디오 장치 간 전환 및 시스템 기본 오디오 장치 설정 제어

## Requirements

### Requirement: 오디오 장치 목록 관리 및 입/출력 구분
시스템은 시스템에 연결된 전체 오디오 장치 목록을 제공하고, 사용자가 원하는 출력 장치와 입력 장치를 짝지어 '장치 프로파일(페어)'을 생성하고 관리할 수 있어야 한다.

#### Scenario: 점선 버튼을 통한 새 프로파일 카드 추가
- **WHEN** 사용자가 장치 탭 하단의 점선 테두리로 된 '프로파일 추가 +' 버튼을 클릭할 때
- **THEN** 새로운 프로파일 카드가 스택에 추가되고 추가 버튼은 아래로 밀려나며, 시스템은 생성된 프로파일을 사용자 설정에 저장해야 한다

#### Scenario: 마우스 오버 시 프로파일 삭제
- **WHEN** 사용자가 특정 프로파일 카드에 마우스를 올리고(Hover), 나타난 휴지통 모양의 삭제 버튼을 클릭할 때
- **THEN** 해당 프로파일 카드가 삭제되고 스택에서 사라지며 사용자 설정에서 제거되어야 한다

### Requirement: 오디오 장치 연결 상태 자동 감지 및 갱신
시스템은 오디오 장치가 새로 연결되거나 연결이 끊어졌을 때 이를 감지하여 장치 목록을 자동으로 갱신해야 한다. (수동 새로고침 버튼 미제공)

#### Scenario: 장치 연결/해제 시 자동 갱신
- **WHEN** 새로운 USB 헤드셋이 시스템에 연결되거나 기존 장치가 해제될 때
- **THEN** 시스템은 즉시 이벤트를 감지하고, 수동 조작 없이 프로파일 내의 장치 선택 콤보박스 목록을 최신 상태로 자동 갱신해야 한다

### Requirement: 장치 순서 커스터마이징
시스템은 사용자가 생성한 '프로파일'들의 순서를 임의로 재배치할 수 있도록 허용해야 한다. 부가적으로, 프로파일 순서 재배치를 위해 드래그 앤 드롭이 진행되는 동안에는 혼동을 방지하기 위해 목록 하단의 '프로파일 추가' 버튼을 숨겨야 한다.

#### Scenario: 그립 핸들을 이용한 드래그 앤 드롭 순서 변경
- **WHEN** 사용자가 프로파일 카드의 좌측 핸들 그립을 마우스로 잡고 드래그하여 순서를 변경할 때
- **THEN** 변경된 프로파일 순서가 사용자 설정에 즉시 저장되고, 트레이 클릭 순환 로직에 반영되어야 한다

#### Scenario: 드래그 시작 시 추가 메뉴 숨김
- **WHEN** 사용자가 특정 프로파일의 좌측 핸들 그립을 잡고 드래그를 시작할 때
- **THEN** 목록 최하단의 '새 프로파일 추가' 영역이 즉시 시각적으로 숨겨져야 한다

#### Scenario: 드래그 종료 시 추가 메뉴 표시
- **WHEN** 프로파일의 드래그 앤 드롭 동작이 종료(완료 또는 취소)될 때
- **THEN** 숨겨졌던 '새 프로파일 추가' 영역이 원래대로 다시 표시되어야 한다

### Requirement: 개별 장치 아이콘 설정
시스템은 사용자가 각 '프로파일'에 대해 커스텀 아이콘을 지정할 수 있도록 해야 한다.

#### Scenario: 프로파일 아이콘 변경
- **WHEN** 사용자가 특정 프로파일의 아이콘 설정 버튼을 눌러 새로운 아이콘 이미지를 지정할 때
- **THEN** 해당 아이콘 경로가 설정에 저장되고, 프로파일이 활성화될 때 트레이 아이콘에 반영되어야 한다

### Requirement: 선호 장치 순환 전환 (사용자 지정 순서)
전환 명령이 발생하면, 시스템은 사용자가 배치한 순서에 따라 다음 '프로파일'에 지정된 입출력 장치로 기본 장치를 변경해야 한다.

#### Scenario: 지정된 순서대로 프로파일 전환
- **WHEN** 사용자가 '헤드셋 프로파일 -> 스피커 프로파일' 순서로 배치한 상태에서 전환 명령이 발생할 때
- **THEN** 활성화 상태가 '헤드셋 프로파일'에서 '스피커 프로파일'로 넘어가고, 해당 프로파일에 설정된 입출력 장치가 모두 기본 장치로 변경되어야 한다

### Requirement: 기본 통신 장치 동시 전환 설정
시스템은 장치 전환 시 '기본 장치'만 변경할지, '기본 통신 장치'도 함께 변경할지 설정 탭에서 옵션을 제공해야 한다.

#### Scenario: 통신 장치 동시 전환
- **WHEN** 사용자가 설정 탭에서 '통신 장치 동시 전환' 옵션을 켜고 트레이 아이콘을 클릭하여 전환할 때
- **THEN** 다음 장치가 시스템의 '기본 재생 장치'와 '기본 통신 장치'로 모두 설정되어야 한다

### Requirement: 즉각적인 장치 전환 알림 제공
시스템은 장치 전환이 발생했을 때 전환된 장치 정보를 담은 시각적 알림(Notification)을 화면에 표시해야 한다. 이 알림은 OS의 기본 큐(Queue)에 의존하지 않고 즉각적으로 나타나야 하며, 사용자가 연속으로 장치를 전환할 경우 알림이 중첩되거나 지연되지 않도록 이전 알림을 즉시 덮어쓰거나 닫아야 한다.

#### Scenario: 단일 장치 전환 시 알림 발생
- **WHEN** 사용자가 트레이 아이콘을 클릭하여 새로운 프로파일로 장치를 전환할 때
- **THEN** 시스템은 딜레이 없이 전환된 장치 이름과 아이콘을 포함한 알림 창을 즉시 표시해야 한다

#### Scenario: 연속적인 장치 전환 시 알림 중복 방지
- **WHEN** 기존 알림 창이 화면에 표시된 상태에서 사용자가 다시 트레이 아이콘을 클릭하여 다른 프로파일로 장치를 연속 전환할 때
- **THEN** 시스템은 기존에 떠 있던 알림 창을 즉시 닫고, 새롭게 전환된 장치 정보가 담긴 알림 창으로 대체하여 화면에 알림이 여러 개 쌓이지 않도록 해야 한다

### Requirement: Windows 11 네이티브 디자인 및 작업 표시줄 동기화
알림 창의 디자인은 Windows 11의 미적 요소(Mica/Acrylic 디자인, 둥근 모서리, 시스템 테마 색상 등)와 일관되게 아름답게 렌더링되어야 한다. 또한 알림 창이 표시되는 위치는 사용자의 작업 표시줄(Taskbar) 위치(상/하/좌/우)를 동적으로 감지하여, 윈도우 기본 알림 토스트와 정확히 동일한 위치에 나타나야 한다.

#### Scenario: 작업 표시줄 위치에 따른 알림 표시
- **WHEN** 사용자의 작업 표시줄이 화면 상단에 위치한 상태에서 알림이 발생할 때
- **THEN** 시스템은 이를 감지하고 우측 상단(또는 순정 윈도우 토스트가 뜨는 정확한 위치)에 네이티브한 디자인의 알림을 표시해야 한다
