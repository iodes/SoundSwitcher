## Requirements

### Requirement: 비활성 장치 전환 대기 (Deferred Activation)
시스템은 사용자가 비활성 상태이거나 연결이 해제된 장치가 포함된 프로파일로 전환을 시도할 때, 에러를 발생시키거나 무시하는 대신 해당 프로파일을 '대기(Pending)' 상태로 등록해야 한다. 시스템은 대상 장치의 상태 변화를 CPU 효율적인 방식(예: Core Audio API의 IMMNotificationClient 이벤트)으로 감지하며 대기해야 한다.

#### Scenario: 비활성 장치 프로파일로 전환 시도
- **WHEN** 사용자가 현재 연결이 끊기거나 비활성 상태인 오디오 장치가 포함된 프로파일로 명시적 전환을 요청할 때
- **THEN** 시스템은 즉시 해당 장치로의 전환 시도를 보류(Defer)하고, 해당 프로파일을 '대기' 상태로 전환하여 이벤트를 모니터링해야 한다.

#### Scenario: 대기 중인 장치 활성화 시 자동 전환
- **WHEN** 대기 상태로 등록된 프로파일의 대상 장치(입력 또는 출력)가 활성화(Active) 상태로 변경될 때
- **THEN** 시스템은 즉시 보류되었던 프로파일 전환 작업을 재개하여 시스템의 기본 오디오 장치를 해당 장치로 변경하고, 대기 상태를 해제해야 한다.

#### Scenario: 여러 장치에 대한 연속된 전환 시도
- **WHEN** 기존에 '대기' 상태인 프로파일이 있는 상황에서 사용자가 다른 프로파일로 전환을 시도할 때
- **THEN** 시스템은 이전의 대기 상태를 취소(초기화)하고, 가장 최근에 요청된 프로파일에 대해서만 즉각적인 전환을 수행하거나 필요시 새로운 대기 상태로 진입해야 한다.
