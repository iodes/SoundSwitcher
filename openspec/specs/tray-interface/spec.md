# Capability: tray-interface

## Purpose
TBD: 시스템 트레이 아이콘 표시, 클릭 이벤트 처리 및 설정용 컨텍스트 메뉴 제공

## Requirements

### Requirement: 시스템 트레이 커스텀 아이콘 표시
애플리케이션이 실행 중일 때 시스템 트레이에 상주하며 현재 활성화된 **프로파일**의 아이콘을 표시해야 한다. 유저가 커스텀 아이콘을 설정한 경우 해당 아이콘이, 그렇지 않은 경우 기본 아이콘이 표시되어야 한다.

#### Scenario: 활성 프로파일 툴팁 및 아이콘 렌더링
- **WHEN** 현재 시스템 장치가 특정 프로파일의 설정과 일치할 때
- **THEN** 트레이 아이콘은 해당 프로파일의 아이콘으로 표시되며, 마우스 오버 시 프로파일 이름이 툴팁으로 표시되어야 한다

#### Scenario: 기본 장치 아이콘 렌더링 (Fallback)
- **WHEN** 현재 활성 장치가 커스텀 아이콘이 설정되지 않은 장치로 전환될 때
- **THEN** 트레이 아이콘 이미지가 시스템에 내장된 Windows 기본 장치 아이콘으로 표시되어야 한다

### Requirement: 한 번의 클릭으로 장치 순환 전환
시스템 트레이 아이콘을 좌클릭하면 즉시 유저가 설정한 순서대로 다음 **프로파일**로 전환되어야 한다. 단, 등록된 프로파일이 0개일 경우 메인 설정 창이 열려야 하며, 등록된 프로파일이 1개일 경우 해당 프로파일이 활성화되도록 하되, 이미 활성화된 상태라면 전환 동작 및 알림을 생략하여 불필요한 순환을 방지해야 한다.

#### Scenario: 좌클릭 프로파일 전환
- **WHEN** 사용자가 트레이 아이콘을 마우스 왼쪽 버튼으로 클릭할 때 (프로파일 2개 이상)
- **THEN** 설정된 순서에 따라 다음 프로파일로 활성화 상태가 넘어가고, 연결된 입/출력 장치가 동시 변경되며, 트레이 아이콘 이미지와 툴팁이 즉시 갱신되어야 한다

#### Scenario: 등록된 프로파일이 없을 때 좌클릭
- **WHEN** 사용자가 트레이 아이콘을 마우스 왼쪽 버튼으로 클릭할 때, 등록된 프로파일이 0개인 경우
- **THEN** 다음 프로파일로 전환을 시도하지 않고 메인 설정 창이 열려야 한다

#### Scenario: 등록된 프로파일이 1개일 때 좌클릭
- **WHEN** 사용자가 트레이 아이콘을 마우스 왼쪽 버튼으로 클릭할 때, 등록된 프로파일이 1개인 경우
- **THEN** 시스템에 현재 해당 프로파일이 활성화되지 않은 상태라면 프로파일을 선택/적용하며, 이미 활성화된 상태라면 불필요한 알림(다음 프로파일 순환)을 띄우지 않고 동작을 종료해야 한다.

### Requirement: 컨텍스트 메뉴 제공
시스템 트레이 아이콘을 우클릭하면 컨텍스트 메뉴가 표시되어야 한다. 이때 컨텍스트 메뉴는 마우스 포인터의 위치를 기준으로 정확하게 렌더링되어야 하며, 아이콘이 숨겨진 트레이 영역(접힘 상태)에 있을 때나 시스템 DPI 스케일이 100%가 아닐 때에도 화면 위치가 어긋나지 않아야 한다. 또한 메뉴에는 '시스템 소리 설정', '설정', '종료' 항목이 포함되어 있어야 하며, '업데이트 확인' 항목은 포함되지 않는다.
메뉴가 표시될 때에는 시스템의 기본 플라이아웃(슬라이드) 애니메이션을 사용하지 않고, 즉시 표시되거나 단순한 페이드 효과만 사용하여 반응성을 높여야 한다. 메뉴 팝업의 시각적 요소로서 주변 요소와 구분되는 적절한 그림자(Drop Shadow) 효과가 일관되게 표시되어야 한다.

#### Scenario: 우클릭 설정 진입
- **WHEN** 사용자가 트레이 아이콘을 우클릭할 때
- **THEN** '시스템 소리 설정', '설정' 및 '종료' 옵션이 포함된 컨텍스트 메뉴가 열려야 하며, '업데이트 확인' 항목은 노출되지 않아야 한다.

#### Scenario: 숨겨진 트레이 아이콘 우클릭 시 컨텍스트 메뉴 위치 보정
- **WHEN** 사용자가 Windows의 '숨겨진 아이콘 표시' 메뉴 내에 위치한 트레이 아이콘을 우클릭할 때
- **THEN** 컨텍스트 메뉴가 잘못된 위치(예: 화면 좌상단 등)가 아닌 마우스 커서의 현재 위치에 정확히 표시되어야 하며, 다른 영역 클릭 시 메뉴가 정상적으로 닫혀야 한다

#### Scenario: 시스템 소리 설정 실행
- **WHEN** 컨텍스트 메뉴에서 '시스템 소리 설정'을 클릭할 때
- **THEN** Windows 기본 소리 제어판(mmsys.cpl) 창이 열려야 한다

#### Scenario: 컨텍스트 메뉴 애니메이션 제한
- **WHEN** 사용자가 트레이 아이콘을 우클릭하여 컨텍스트 메뉴가 열릴 때
- **THEN** 메뉴가 슬라이드/플라이아웃 애니메이션 없이 즉시 표시되거나 페이드 효과로만 나타나야 한다

#### Scenario: 컨텍스트 메뉴 테두리 그림자 표시
- **WHEN** 사용자가 트레이 아이콘을 우클릭하여 컨텍스트 메뉴가 열릴 때
- **THEN** 컨텍스트 메뉴의 외곽 테두리에 WPF의 기본 컨텍스트 메뉴와 시각적으로 일치하는 수준의 그림자 효과가 적용되어 애니메이션과 충돌 없이 온전히 표시되어야 한다.
